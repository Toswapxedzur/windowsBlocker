# Vault デスクトップ アプリの機能リファレンス

## 目的と境界

これは、Vault デスクトップ アプリケーション インターフェイスの信頼できるリファレンスです。これは、Vault ブラウザ拡張マニュアルとは意図的に分離されています。

デスクトップ アプリは **ネイティブ アプリケーションとアプリケーション ウィンドウ**を管理します。ブラウザ拡張機能は、Web サイト、ブラウザ タブ、およびサポートされている Web プラットフォーム フィードを管理します。これらは、グループ、スケジュール、タイマー、フリーズ、スヌーズ、カスタム ルール、およびオプションのブリッジなど、同じアイデアを共有していますが、同じ適用面を持っていません。

このドキュメントを使用して、デスクトップ アプリの動作を構成、監査、再現、または保守します。実装とこのマニュアルが異なる場合、コードは正規です。

## 1. デスクトップ アプリが制御できることと制御できないこと

Vault は、選択されたネイティブ アプリケーションのフォーカス ポリシーを評価します。ネイティブの強制機能が利用可能な場合、一致するアプリケーション ターゲットに最新の計画を適用し、シールド/ステータスの結果をホスト UI に報告できます。

次のことが可能です。

- グループの作成、有効化、無効化、並べ替え、インポート、エクスポート、フリーズ、スヌーズ、および削除。
- アプリケーション ピッカーを通じて選択されたターゲット ネイティブ アプリケーション。
- 即時ブロック、時間指定の許可、またはカウントアップのみのタイマーを適用します。
- 通常のグループを平日および現地時間枠に制限します。
- アプリケーションのライフサイクル イベントに対してカスタム JavaScript ポリシー ルールを実行します。
- ルールによって作成されたネイティブ ステータス/パネル情報をホスト経由で表示します。
- サポートされているカスタム ルール ファイル リクエスト用のオプションのローカル フォルダーを管理します。
- ローカル Vault ブリッジを介して、互換性のある明示的にリンクされたグループに参加します。

以下のことはできません。

- ブラウザ拡張機能として機能し、Web サイト DOM を検査したり、ブラウザ フィード カードを操作したりします。
- オペレーティング システムがすべてのアプリケーション、プロセス、ウィンドウ、またはシステム サービスを制御できることを保証します。
- アプリケーションの選択をリモート管理、デバイス監視、またはファイアウォールに変えます。
- DOM、ナビゲーション、リダイレクト、タブ コントロールなどのブラウザー専用のカスタム ヘルパーをネイティブ ランタイムで動作させます。
- ローカル ブリッジが実行されているため、すべてのグループが自動的に同期されます。

## 2. 語彙

|用語 |意味 |
| --- | --- |
|グループ |名前付きフォーカス ポリシー オブジェクト。グループ名は、現在の Vault エンドポイント内で一意である必要があります。 |
|ターゲット |グループに対して選択されたネイティブ アプリケーション ID。 |
|デフォルトのアプリケーショングループ |アプリ ピッカーからのネイティブ アプリケーションをターゲットとする通常のグループ。 |
| Custom group | A group whose JavaScript rule reacts to application-policy events. |
|一致 |現在のフォアグラウンド/実行中のアプリケーションは、有効かつアクティブなグループ ターゲットまたはカスタム ルール条件と一致します。 |
|アクティブ |通常のスケジュール内で有効になり、アクティブにスヌーズされません。 |
|施行計画 |ネイティブ ホストが該当するグループを評価した後の許可/シールド/ステータスの決定。 |
|フリーズ |グループの通常の変更に対する保護。 |
|スヌーズ |通常のグループ ポリシーからの一時的な例外。 |

## 3. ターゲット ID とアプリケーション ピッカー

デフォルト アプリケーション グループの **+** ピッカーを使用してアプリケーションを選択します。 Vault には、正規化された ID と表示名が保存されます。

|ホスト |照合に使用されるターゲット ID |
| --- | --- |
| macOS |利用可能な場合はアプリケーション バンドルの識別子。 |
|ウィンドウズ |アプリケーション ピッカーによって提供される正規化された実行可能ファイルのパスまたはプロセス名。 |

表示名はエディター用です。正規化された値は、ネイティブ強制レイヤーによって使用される ID です。 UI でアプリケーションの名前を変更しても ID は変更されません。ターゲットは、カスタム ルール ポリシーで使用するためのタグを保持することもできます。

アプリケーション ターゲット フィールドに Web サイトの URL を入力し、ネイティブ アプリケーションの強制を期待しないでください。 Web サイトをブロックするには、拡張機能のサイト グループを使用します。

## 4. グループのライフサイクルと優先順位

新しいグループはデフォルトで有効になります。グループ リストは、選択、有効化/無効化、ドラッグ順序、追加、クリア、インポート、エクスポート、および削除をサポートします。選択したグループがエディタで開きます。

通常のフィールド編集は、エディターの自動保存ポリシーを通じて保存されます。グループを凍結すると、通常の編集コントロールが無効になります。カスタム ソースは異なります。テキストを保存してもアクティブになりません。 **実行** は、現在のソースをポリシー ランタイムにロードする操作です。

複数のグループが同じアプリケーションに一致する可能性があります。 Vault は、保存された順序でグループ ポリシーを評価し、ネイティブの適用計画を構築します。特にグループが異なる時間指定ポリシーを使用している場合や、カスタム ルールが許可/シールドの決定を発行する場合は、意図的に重複を維持します。グループの順序を変更して、意図した優先順位を明確にします。競合する構成が特定のユーザーフレンドリーな方法で解決されることに依存しないでください。

## 5. 通常のアプリケーショングループ

### 5.1 グループの状態

|フィールド |機能契約 |
| --- | --- |
|名前 |このエンドポイント内では空ではなく、トリミングされ、大文字と小文字を区別せずに一意です。 |
|有効 |無効化されたグループは保持されますが、通常の強制には参加しません。 |
|ターゲット |ピッカーから選択された 1 つ以上のアプリケーション ID。 |
|行動 |即時ブロック、許容値後のブロック、またはタイマー/カウントアップ。 |
|スケジュール |選択された平日とオプションの現地時間ウィンドウ。 |
|フリーズ |なし、冷凍、完全冷凍、または保護者による冷凍。 |
|スヌーズ |グループごとの一時例外ポリシー。 |
|フォールバック/ステータス メッセージ |ネイティブ ホストがシールド/ステータス応答を適用するときに表示できるメッセージ。 |

空の Default グループにはアプリケーション ターゲットが選択されていないため、単に存在するだけではアプリケーションと一致しません。

### 5.2 ブロック動作

|行動 |結果 |
| --- | --- |
|すぐにブロック |アクティブなターゲットが一致すると、即座にネイティブ ブロック/シールドが決定されます。 |
|数分後にブロック |グループ手当に対してマッチング使用が発生します。許容量がなくなると、グループは、使用期間がリセットされるか、別の状態でグループが非アクティブになるまで、ネイティブ ブロック/シールドの決定を生成します。 |
|タイマー (カウントアップ、ブロックなし) |一致する使用が測定され、表示される場合がありますが、そのタイマーだけではブロックが生成されることはありません。 |

新しいグループでは、変更しない限り、15 分の許容時間と 24 時間のリセット間隔が使用されます。期限付き使用はグループに属しているため、一致するすべてのターゲットがそのグループ ポリシーを共有します。ブロックに対する正確な応答はネイティブ ホストによって実装され、オペレーティング システムのアクセス許可とサポートされている強制メカニズムによって制限されます。

### 5.3 スケジュール

スケジュールは通常のグループに適用されます。カスタム グループは、JavaScript で独自の時間を決定します。

月曜日から日曜日までの任意の組み合わせを選択します。各時間ウィンドウは現地時間で 1 行です。

```text
0900-1200
1330-1730
```

正確に受け入れられる形式は HHMM-HHMM です。時間は 00 から 23、分は 00 から 59 でなければならず、開始は同日の終了より早くなければなりません。ウィンドウには開始部分が含まれ、終了部分は含まれません。深夜をまたぐ窓口は受け付けておりません。空のウィンドウは、選択した日全体を意味します。

通常グループは、次の場合にのみアクティブになります。

1. 有効になっています。
2. 現在の曜日が選択されます。
3. 現地時間が設定されたウィンドウ内にあるか、グループにウィンドウがありません。
4. アクティブなスヌーズ状態ではありません。

### 5.4 スヌーズ

スヌーズは、アクティブな強制から通常のグループを一時的に削除します。そのフェーズは次のとおりです。

|フェーズ |結果 |
| --- | --- |
|保留中 |リクエストは存在しますが、アクティベーション遅延が経過していません。グループは活動を続けています。 |
|アクティブ |グループは、スヌーズ期間中、一時的に非アクティブになります。 |
|クールダウン |スヌーズが終了し、グループは再びアクティブになりましたが、新しいスヌーズはまだ利用できません。 |

|設定 |ルール |
| --- | --- |
|スヌーズを許可する |オフの場合、グループは通常スヌーズできません。 |
|スヌーズ時間 |正の分数。新しいグループのデフォルトは 30 分です。 |
|アクティベーションの遅延 |スヌーズがアクティブになる 0 分以上前。 |
|クールダウン |アクティブなスヌーズが終了してから 0 ～ 5 分後。 |
|確認事項 |必要な確認インタラクションの非負の整数。 |

アクティブなスヌーズは一時的なポリシーの例外であり、削除や凍結解除ではありません。グループ構成はそのまま残ります。

### 5.5 フリーズ

フリーズは意図的な変更バリアです。

|モード |契約 |
| --- | --- |
|冷凍 |通常の編集と通常の状態変更は、製品の凍結解除確認フローが成功するまでロックされたままになります。 |
|厳密冷凍 |グループは、完全凍結期間が終了するまで凍結を解除できません。期間は正であり、72 時間に制限されます。 |
|凍結された親 |凍結/凍結解除アクションにはガーディアンパスワード管理が必要です。 |

エディターでモードを選択しても、グループ自体はフリーズしません。適用するにはフリーズアクションを使用します。ブリッジリンクされたグループは、必要なメンバーがオフラインの間、調整されたフリーズ制御をロックすることもできます。

## 6. ネイティブ強制とデバイス制御

エディターは、オペレーティング システムがグループを強制する機能を許可していない場合でも、グループを正確に保存できます。権限を変更した後は、**[設定] → [デバイス コントロール]** とライブ ネイティブ ステータスを必ず確認してください。

ネイティブ ホストは、現在のオペレーティング システム、アプリケーション、ウィンドウ、およびアクセス許可の状態に対してどのアクションが可能かを決定します。次の場合、ルールは正しく構成されていても目に見える効果がありません。

- デバイス制御が許可されていないか、取り消されています。
- グループが無効になっているか、スケジュールされていないか、アクティブにスヌーズされています。
- フォーカスされたプロセスが、選択された正規化されたターゲットと一致しません。
- オペレーティング システムは、そのターゲットに対するアクションを拒否します。
- 調整された状態を必要とするアクションのブリッジ依存関係はオフラインです。

トーストの保存に成功したことを、アクティブな強制が利用できることの証拠として扱わないでください。グループがアクティブな間に選択したターゲットをテストし、ホストのステータスを検査します。

## 7. カスタム グループとネイティブ ポリシー ルール

カスタム グループは、ネイティブ JavaScript ポリシー ランタイムで実行されます。これらはブラウザのカスタム ルールではありません。ブラウザーの DOM、タブ、ナビゲーション、URL リダイレクト、およびフィード制御の動作は意図的に使用できません。

### 7.1 ソースのライフサイクル

Use a function expression:

```js
(events, helpers) => {
  events.on("focusEvent", "shield-focus", (event, h) => {
    if (event.target?.id) event.setResult(-1);
  });
}
```

Run loads the source and its event registrations. Running again unloads the old source and resets its rule-owned handlers, timers, panels, persistence, and dynamic app blocklist. A source that does not evaluate to a function cannot be loaded.

### 7.2 ネイティブの組み込みイベント

|イベント |意味 |
| --- | --- |
|ティックイベント |定期的なホスト ティック。 intervalMs 登録オプションにより、ハンドラーのレート制限を行うことができます。 |
|タイマー終了 |ルール所有のカウントダウンがゼロに達します。 |
|スヌーズプレス |ユーザーがカスタム グループの [スヌーズ開始] を押します。 |
|パネルイベント |カスタムパネルコントロールが使用されます。 |
|ローカルファイルイベント |要求されたローカル フォルダー アクションが完了します。 |
|オープンアプリイベント |追跡されたアプリケーションが開きます。 |
|閉じるアプリイベント |追跡されているアプリケーションが終了します。 |
|フォーカスイベント |フォアグラウンド アプリケーションがアプリケーションに変わります。 |
|アンフォーカスイベント |フォアグラウンド アプリケーションがアプリケーションから変化します。 |
|最小化イベント / unminimizeEvent |ホストは、サポートされているウィンドウ最小化遷移を報告します。 |
|スイッチアプリイベント |フォアグラウンド アプリケーションは、あるアプリから別のアプリに変わります。 |
|アプリ変更イベント |一般的なアプリケーションのライフサイクル/変更イベント。 |

イベント オブジェクトには、タイプ、groupId/groupID、groupName、相当する URL/ホスト名、時間、データ、およびターゲットが含まれます。ネイティブ アプリケーションの場合、ターゲットは、フォーカス ターゲットが構成されたターゲットと一致する場合に、ID、種類、表示名、正規化された値、およびタグを公開します。

アプリケーション ライフサイクル イベント データには、現在のアプリ ID/名前、グループ名、シリアル化された実行中のアプリのスナップショット、および BundleId、previousAppId、currentAppId、変更理由などのイベント固有の値が含まれます。

### 7.3 イベント API と決定

レジストリには、on/register、off/unregister、unregisterAll、countRegistered、getEvent、および getEvents が用意されています。優先度の高いものが最初に実行されます。優先度が等しい場合は、登録順序が維持されます。レジストリにはグループごとのハンドラーの上限があります。

イベント オブジェクトは以下をサポートします。

|方法 |結果 |
| --- | --- |
| setResult(-1) |ネイティブのシールド/ブロックの決定を生成します。デスクトップ ルールにはブラウザのリダイレクト ターゲットがないため、文字列の結果もネイティブ ブロックになります。 |
| allow(reason) または setResult(1) |イベントの許可決定を生成します。 |
| setShieldMessage(メッセージ) |ネイティブ ブロックの人向きのシールド/ステータス メッセージを設定します。 |
| stopPropagation() |現在のイベントの後のハンドラーを停止します。 |
|ブロック (アプリ ID)、ブロック解除 (アプリ ID) |動的ネイティブ アプリケーション ブロックを追加/削除します。 |
|閉じる(appId)、開く(appId) |サポートされているネイティブのクローズ/オープン アクションをリクエストします。 |
| post(タイプ, データ) |ネイティブ ランタイム内でネストされたカスタム イベントをディスパッチします。 |

アプリ ランタイムでは、タイマー、永続化、パネル、ログ、ローカル フォルダーの操作、アプリケーション ウィンドウ ヘルパー、および URL 分類子ユーティリティが可能です。 DOM、ナビゲーション、リダイレクト、およびブラウザタブヘルパーを意図的に利用できない/不活性なものとして扱います。

### 7.4 ネイティブヘルパー

|ヘルパー |ネイティブの動作 |
| --- | --- |
|ログヘルパー |アプリ/ポップアップ/画面ログの決定を出力します。 |
|取得タイマーヘルパー |境界、ステップ、スコープ/ドメイン述部、一時停止/再開、状態検査、および timerEnded 遷移を備えた前方/後方タイマーを作成します。タイマーはそれ自体をシールドしません。 |
| getPersistenceHelper |グループごとの JSON 状態: 取得、設定、削除、has、キー、エントリ、クリア、サイズ。 |
|ストレージヘルパーを取得する |永続性とホストの非同期要求プレースホルダー。同期外部応答を想定しないでください。 |
|取得ウィンドウヘルパー |現在/実行中のアプリケーションを読み取り、アプリケーションの閉じる/開く/ブロック/ブロック解除のアクションを要求します。 |
|パネルヘルパー |検証済みのネイティブ パネル スナップショット、コントロール、インライン ハンドラー、およびpanelEvent リアクションを作成します。 |
|ローカルフォルダヘルパー |キューでは、ユーザーが付与した root での相対的な .txt、.csv、および .json 操作が許可されました。完了はlocalFileEventです。 |
| getDomainHelper / getDomainUtility | URL のような値についても推論するルールの URL およびプラットフォーム分類子。 |
| getPlatformHelper / プラットフォーム | URL 分類子は引き続き利用可能です。デスクトップ ホストには Web サイト DOM がないため、ネイティブ フィード/DOM 制御呼び出しは不活性になります。 |

カスタム パネルは、ブラウザー ランタイムと同じ宣言型コントロール ボキャブラリー (テキスト、チェックボックス、選択、テキスト入力、テキストエリア、ボタン、セクション、タイマー、numberInput、範囲、トグル、ラジオ、日付、時刻、色、ピン、およびサニタイズされた HTML) を使用します。ネイティブ ホストは、現在のプラットフォームで表示できるパネルの量を決定します。

## 8. ローカルファイルフォルダー

ローカル ファイル フォルダーは、カスタム ルールのユーザー許可のオプションの境界です。ルールでは、テキスト/CSV/JSON の読み取り、書き込み、追加、リスト、存在テスト、および JSON 操作をリクエストできます。パスは常に選択したルートを基準とします。絶対パス、トラバーサル セグメント、非表示のパス コンポーネント、サポートされていない拡張子、およびルート外の操作は拒否されます。

ルールでフォルダーが必要なくなった場合は、フォルダーを取り消します。ルールは、利用できない権限と失敗した localFileEvent の結果を処理する必要があります。選択したフォルダーが再起動後も許可されたままであると想定してはなりません。

## 9. Web アプリブリッジ

ブリッジは、互換性のある Vault プログラム間のオプションのローカル同期です。ネイティブ デスクトップ アプリはローカル ハブをホストできます。クライアントはサポートされているローカル アドレスで接続します。

接続状態は、オフ、接続中、切断、接続/実行中、およびエラーです。プログラムを接続しても、すべてのグループが結合されるわけではありません。ユーザーは、適格な一致グループを明示的にリンクする必要があります。

グループリンクの場合:

1. [設定] でネイティブ ハブを起動します。
2. 他の互換性のある Vault エンドポイントに接続します。
3. 同じ名前とタイプで、一致する凍結解除されたグループを作成します。
4. グループブリッジセクションでプログラムを選択し、グループを接続します。

リンクされたグループはクラスターを形成します。サポートされている共通ポリシー値、使用法、およびスヌーズ状態は、メンバーが接続されている間に同期できます。切断すると同期が一時停止され、ローカル グループが保持されます。ブラウザのみのターゲット、サポートされていないカスタム アクション、およびプラットフォーム固有のフィールドは転送が保証されません。

## 10. インポート、エクスポート、リセット、監査

エクスポートでは、互換性のあるグループ表現が 1 つ保存されます。インポートは、互換性のあるグループ データを検証/正規化し、ローカル名の一意性を強制します。グループの削除は、選択したグループとそれに関連付けられた状態を削除します。 [クリア] は、確認後にすべてのグループを削除します。デフォルトにリセットすると、グローバル エディターの設定に影響します。最初に保持する必要があるものはすべてエクスポートします。

デスクトップ ルールに依存する前に:

1. デバイス制御が許可されていることを確認します。
2. 選択したターゲットの正規化された ID を確認します。
3. 有効な状態、スケジュール、フリーズ状態、およびスヌーズ段階を確認します。
4. 即時動作、時間指定動作、およびカウントアップ動作を個別にテストします。
5. カスタム グループの場合、正確なソースを実行し、登録された各アプリ イベントをテストします。
6. ローカル フォルダーの失敗と操作の成功を確認します。
7. グループがリンクされている場合、ブリッジのオフライン/接続動作を確認します。
