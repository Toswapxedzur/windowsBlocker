# Vault 데스크탑 앱 기능 참조

## 목적과 경계

이는 Vault 데스크탑 응용프로그램 인터페이스에 대한 신뢰할 수 있는 참조입니다. 이는 의도적으로 Vault 브라우저 확장 매뉴얼과 별개입니다.

데스크톱 앱은 **기본 애플리케이션과 애플리케이션 창**을 관리합니다. 브라우저 확장은 웹사이트, 브라우저 탭 및 지원되는 웹 플랫폼 피드를 관리합니다. 그룹, 일정, 타이머, 정지, 다시 알림, 사용자 지정 규칙, 선택적 브리지 등 동일한 아이디어를 공유하지만 적용 표면은 동일하지 않습니다.

이 문서를 사용하여 데스크톱 앱 동작을 구성, 감사, 재현 또는 유지 관리하세요. 구현과 이 매뉴얼이 다른 경우 코드는 표준입니다.

## 1. 데스크톱 앱이 제어할 수 있는 것과 제어할 수 없는 것

Vault는 선택된 기본 애플리케이션에 대한 중점 정책을 평가합니다. 기본 적용 기능을 사용할 수 있는 경우 현재 계획을 일치하는 애플리케이션 대상에 적용하고 보호/상태 결과를 호스트 UI에 보고할 수 있습니다.

다음을 수행할 수 있습니다.

- 그룹 생성, 활성화, 비활성화, 재정렬, 가져오기, 내보내기, 고정, 일시 중지 및 제거
- 애플리케이션 선택기를 통해 선택된 기본 애플리케이션을 대상으로 합니다.
- 즉시 차단, 시간 제한 허용 또는 카운트업 전용 타이머를 적용합니다.
- 일반 그룹을 평일 및 현지 시간 창으로 제한합니다.
- 애플리케이션 수명주기 이벤트에 대한 사용자 정의 JavaScript 정책 규칙을 실행합니다.
- 호스트를 통해 규칙이 생성한 기본 상태/패널 정보를 표시합니다.
- 지원되는 사용자 정의 규칙 파일 요청을 위한 선택적 로컬 폴더를 관리합니다.
- 로컬 Vault 브리지를 통해 명시적으로 연결된 호환 가능한 그룹에 가입하세요.

다음은 불가능합니다:

- 브라우저 확장 기능을 수행하거나, 웹사이트 DOM을 검사하거나, 브라우저 피드 카드를 조작합니다.
- 운영 체제가 모든 애플리케이션, 프로세스, 창 또는 시스템 서비스를 제어할 수 있도록 보장합니다.
- 애플리케이션 선택을 원격 관리, 장치 감시 또는 방화벽으로 전환합니다.
- DOM, 탐색, 리디렉션 또는 탭 제어와 같은 브라우저 전용 사용자 정의 도우미가 기본 런타임에서 작동하도록 합니다.
- 로컬 브리지가 실행 중이기 때문에 모든 그룹을 자동으로 동기화합니다.

## 2. 어휘

| 기간 | 의미 |
| --- | --- |
| 그룹 | 명명된 포커스 정책 개체입니다. 그룹 이름은 현재 Vault 엔드포인트 내에서 고유해야 합니다. |
| 대상 | 그룹에 대해 선택된 기본 애플리케이션 ID입니다. |
| 기본 애플리케이션 그룹 | 앱 선택기의 기본 애플리케이션을 대상으로 하는 일반 그룹입니다. |
| Custom group | A group whose JavaScript rule reacts to application-policy events. |
| 경기 | 현재 포그라운드/실행 중인 애플리케이션은 활성화된 활성 그룹 대상 또는 사용자 정의 규칙 조건과 일치합니다. |
| 활성 | 활성화되고 정상적인 일정 내에서 활성화되며 적극적으로 다시 알림이 설정되지 않습니다. |
| 시행계획 | 해당 그룹을 평가한 후 기본 호스트의 허용/보호/상태 결정입니다. |
| 동결 | 그룹의 일반적인 수정으로부터 보호합니다. |
| 스누즈 | 일반 그룹 정책의 일시적인 예외입니다. |

## 3. 대상 ID 및 애플리케이션 선택기

기본 애플리케이션 그룹의 **++** 선택기를 통해 애플리케이션을 선택합니다. Vault는 정규화된 ID와 표시 이름을 저장합니다.

| 호스트 | 매칭에 사용되는 대상 ID |
| --- | --- |
| macOS | 가능한 경우 애플리케이션 번들 식별자입니다. |
| 윈도우 | 애플리케이션 선택기에서 제공하는 정규화된 실행 파일 경로 또는 프로세스 이름입니다. |

표시 이름은 편집자의 이름입니다. 정규화된 값은 기본 적용 계층에서 사용되는 ID입니다. UI에서 애플리케이션 이름을 바꿔도 ID는 변경되지 않습니다. 대상은 사용자 지정 규칙 정책 사용을 위한 태그를 전달할 수도 있습니다.

애플리케이션 대상 필드에 웹사이트 URL을 입력하지 말고 기본 애플리케이션 적용을 기대하세요. 웹사이트 차단을 위해 확장 프로그램의 사이트 그룹을 사용하세요.

## 4. 그룹 수명주기 및 우선순위

새 그룹은 기본적으로 활성화됩니다. 그룹 목록은 선택, 활성화/비활성화, 드래그 순서, 추가, 지우기, 가져오기, 내보내기 및 삭제를 지원합니다. 선택한 그룹이 편집기에서 열립니다.

일반 필드 편집은 편집기의 자동 저장 정책을 통해 저장됩니다. 고정된 그룹은 일반 편집 컨트롤을 비활성화합니다. 사용자 정의 소스는 다릅니다. 텍스트를 저장해도 활성화되지 않습니다. **실행**은 현재 소스를 정책 런타임에 로드하는 작업입니다.

여러 그룹이 동일한 애플리케이션과 일치할 수 있습니다. Vault는 저장된 순서대로 그룹 정책을 평가하고 기본 시행 계획을 수립합니다. 특히 그룹이 서로 다른 시간 제한 정책을 사용하거나 사용자 정의 규칙이 허용/차단 결정을 내리는 경우 중복을 의도적으로 유지하십시오. 의도한 우선순위를 명확하게 하기 위해 그룹 순서를 변경합니다. 특정 사용자 친화적인 방식으로 해결되는 충돌 구성에 의존하지 마십시오.

## 5. 일반 응용 프로그램 그룹

### 5.1 그룹 상태

| 필드 | 기능 계약 |
| --- | --- |
| 이름 | 이 끝점 내에서 비어 있지 않고 잘리고 대소문자를 구분하지 않고 고유합니다. |
| 활성화됨 | 비활성화된 그룹은 유지되지만 정상적인 시행에는 참여하지 않습니다. |
| 대상 | 선택기에서 선택된 하나 이상의 애플리케이션 ID입니다. |
| 행동 | 즉시 차단, 허용 후 차단 또는 타이머/카운트업. |
| 일정 | 선택된 평일 및 선택적인 현지 시간 창. |
| 동결 | 없음, 냉동, 엄격 냉동 또는 부모 냉동. |
| 스누즈 | 그룹별 임시 예외 정책. |
| 대체/상태 메시지 | 기본 호스트가 보호/상태 응답을 적용할 때 표시할 수 있는 메시지입니다. |

빈 기본 그룹에는 선택된 응용 프로그램 대상이 없으므로 단순히 존재하는 것만으로는 응용 프로그램과 일치하지 않습니다.

### 5.2 차단 동작

| 행동 | 결과 |
| --- | --- |
| 즉시 차단 | 일치하는 활성 대상은 즉각적인 기본 차단/방패 결정을 내립니다. |
| 몇 분 후에 차단 | 그룹 수당에 대해 상응하는 사용이 발생합니다. 허용량이 소진되면 그룹은 사용 기간이 재설정되거나 다른 상태에서 그룹을 비활성화할 때까지 기본 차단/보호 결정을 내립니다. |
| 타이머(카운트업, 차단 없음) | 일치하는 사용량이 측정되어 표시될 수 있지만 해당 타이머만으로는 블록이 생성되지 않습니다. |

새 그룹은 변경되지 않는 한 15분의 허용 시간과 24시간의 재설정 간격을 사용합니다. 시간 초과 사용은 그룹에 속하므로 일치하는 모든 대상은 해당 그룹 정책을 공유합니다. 블록에 대한 정확한 응답은 기본 호스트에 의해 구현되며 운영 체제 권한 및 지원되는 시행 메커니즘에 의해 제한됩니다.

### 5.3 일정

일정은 일반 그룹에 적용됩니다. 사용자 정의 그룹은 JavaScript에서 자체적으로 시간을 결정합니다.

월요일부터 일요일까지의 조합을 선택하세요. 각 시간 창은 현지 시간의 한 줄입니다.

```text
0900-1200
1330-1730
```

허용되는 정확한 형식은 HHMM-HHMM입니다. 시간은 00~23분, 00~59분이어야 하며, 시작은 같은 날 종료보다 빨라야 합니다. 창은 시작을 포함하고 끝을 제외합니다. 자정 이후 창구는 허용되지 않습니다. 빈 창은 선택한 날짜 전체를 의미합니다.

일반 그룹은 다음과 같은 경우에만 활성화됩니다.

1. 활성화되어 있습니다.
2. 현재 요일이 선택되었습니다.
3. 현지 시간이 구성된 기간 내에 있거나 그룹에 기간이 없습니다.
4. 활성 스누즈 상태가 아닙니다.

### 5.4 스누즈

다시 알림은 활성 시행에서 일반 그룹을 일시적으로 제거합니다. 그 단계는 다음과 같습니다:

| 단계 | 결과 |
| --- | --- |
| 보류 중 | 요청이 존재하지만 활성화 지연이 경과되지 않았습니다. 그룹은 활성 상태로 유지됩니다. |
| 활성 | 일시중지 기간 동안 그룹이 일시적으로 비활성화됩니다. |
| 쿨다운 | 다시 알림이 종료되고 그룹이 다시 활성화되지만 아직 새 다시 알림을 사용할 수 없습니다. |

| 설정 | 규칙 |
| --- | --- |
| 일시 중지 허용 | 꺼져 있으면 그룹은 일반적으로 다시 알림을 받을 수 없습니다. |
| 스누즈 기간 | 양수(분)입니다. 새 그룹의 기본값은 30분입니다. |
| 활성화 지연 | 다시 알림이 활성화되기까지 0분 이상 소요됩니다. |
| 쿨다운 | 활성 다시 알림이 끝난 후 0~5분입니다. |
| 확인 | 필수 확인 상호작용의 정수가 아닌 정수입니다. |

활성 다시 알림은 삭제나 고정 해제가 아닌 임시 정책 예외입니다. 그룹 구성은 그대로 유지됩니다.

### 5.5 동결

동결은 의도적인 수정 장벽입니다.

| 모드 | 계약 |
| --- | --- |
| 냉동 | 일반 편집 및 일반 상태 변경은 제품의 고정 해제 확인 흐름이 성공할 때까지 잠긴 상태로 유지됩니다. |
| 엄격한 냉동 | 엄격한 고정 기간이 끝나기 전에는 그룹을 고정 해제할 수 없습니다. 지속 시간은 양수이며 72시간으로 제한됩니다. |
| 부모의 냉동 | 고정/고정 해제 작업에는 보호자 비밀번호 관리가 필요합니다. |

편집기에서 모드를 선택해도 그룹 자체가 고정되지는 않습니다. 그것을 적용하려면 동결 동작을 사용하십시오. 브리지에 연결된 그룹은 필요한 구성원이 오프라인인 동안 조정된 동결 제어를 잠글 수도 있습니다.

## 6. 기본 적용 및 장치 제어

운영 체제에서 그룹을 적용하는 기능을 부여하지 않은 경우에도 편집자는 그룹을 정확하게 저장할 수 있습니다. 권한을 변경한 후에는 항상 **설정 → 장치 제어** 및 라이브 네이티브 상태를 확인하세요.

기본 호스트는 현재 운영 체제, 애플리케이션, 창 및 권한 상태에 대해 가능한 작업을 결정합니다. 다음과 같은 경우 규칙이 올바르게 구성되었으나 눈에 띄는 효과가 없을 수 있습니다.

- 장치 제어가 승인되지 않았거나 취소되었습니다.
- 그룹이 비활성화되었거나 예약되었거나 적극적으로 일시 중지되었습니다.
- 집중된 프로세스가 선택된 정규화된 목표와 일치하지 않습니다.
- 운영 체제가 해당 대상에 대한 작업을 거부합니다.
- 조정된 상태가 필요한 작업에 대해 브리지 종속성이 오프라인 상태입니다.

성공적인 저장 알림을 활성 적용이 가능하다는 증거로 처리하지 마십시오. 그룹이 활성화된 동안 선택한 대상을 테스트하고 호스트 상태를 검사합니다.

## 7. 사용자 지정 그룹 및 기본 정책 규칙

사용자 지정 그룹은 기본 JavaScript 정책 런타임에서 실행됩니다. 이는 브라우저 사용자 정의 규칙이 아닙니다. 브라우저 DOM, 탭, 탐색, URL 리디렉션 및 피드 제어 동작은 의도적으로 사용할 수 없습니다.

### 7.1 소스 수명주기

Use a function expression:

```js
(events, helpers) => {
  events.on("focusEvent", "shield-focus", (event, h) => {
    if (event.target?.id) event.setResult(-1);
  });
}
```

Run loads the source and its event registrations. Running again unloads the old source and resets its rule-owned handlers, timers, panels, persistence, and dynamic app blocklist. A source that does not evaluate to a function cannot be loaded.

### 7.2 네이티브 내장 이벤트

| 이벤트 | 의미 |
| --- | --- |
| 틱이벤트 | 주기적인 호스트 틱. IntervalMs 등록 옵션은 핸들러의 속도를 제한할 수 있습니다. |
| 타이머 종료됨 | 규칙 소유 카운트다운이 0에 도달합니다. |
| 스누즈프레스 | 사용자가 사용자 정의 그룹에 대해 스누즈 시작을 누릅니다. |
| 패널이벤트 | 사용자 정의 패널 컨트롤이 사용됩니다. |
| 로컬파일이벤트 | 요청된 로컬 ​​폴더 작업이 완료됩니다. |
| 오픈앱이벤트 | 추적된 애플리케이션이 열립니다. |
| closeApp이벤트 | 추적된 애플리케이션이 닫힙니다. |
| 포커스이벤트 | 포그라운드 애플리케이션이 애플리케이션으로 변경됩니다. |
| 언포커스이벤트 | 포그라운드 애플리케이션이 애플리케이션에서 벗어나 변경됩니다. |
| MinimizeEvent / unminimizeEvent | 호스트는 지원되는 창 최소화 전환을 보고합니다. |
| 스위치앱이벤트 | 포그라운드 애플리케이션은 한 앱에서 다른 앱으로 변경됩니다. |
| 앱변경이벤트 | 일반 애플리케이션 수명주기/변경 이벤트. |

이벤트 객체에는 유형, groupId/groupID, groupName, URL/호스트 이름 등가물, 시간, 데이터 및 대상이 포함됩니다. 네이티브 애플리케이션의 경우 포커스 대상이 구성된 대상과 일치하면 대상은 ID, 종류, 디스플레이 이름, 정규화된 값 및 태그를 노출합니다.

애플리케이션 수명 주기 이벤트 데이터에는 현재 앱 ID/이름, 그룹 이름, 직렬화된 실행 중인 앱 스냅샷 및 이벤트별 값(예: BundleId, PreviousAppId, currentAppId 또는 변경 이유)이 포함됩니다.

### 7.3 이벤트 API 및 결정

레지스트리는 on/register, off/unregister, unregisterAll, countRegistered, getEvent 및 getEvents를 제공합니다. 우선순위가 높을수록 먼저 실행됩니다. 동일한 우선순위는 등록 순서를 유지합니다. 레지스트리에는 그룹별 처리기 한도가 있습니다.

이벤트 객체는 다음을 지원합니다.

| 방법 | 결과 |
| --- | --- |
| setResult(-1) | 네이티브 쉴드/블록 결정을 생성합니다. 데스크톱 규칙에는 브라우저 리디렉션 대상이 없기 때문에 문자열 결과도 기본 차단이 됩니다. |
| 허용(이유) 또는 setResult(1) | 이벤트에 대한 허용 결정을 생성합니다. |
| setShieldMessage(메시지) | 기본 블록에 대해 사람을 향한 방패/상태 메시지를 설정합니다. |
| 중지 전파() | 현재 이벤트에 대한 이후 처리기를 중지합니다. |
| 차단(appId), 차단 해제(appId) | 동적 네이티브 애플리케이션 블록을 추가/제거합니다. |
| 닫기(appId), 열기(appId) | 지원되는 기본 닫기/열기 작업을 요청하세요. |
| 게시물(유형, 데이터) | 네이티브 런타임 내에서 중첩된 맞춤 이벤트를 전달합니다. |

앱 런타임은 타이머, 지속성, 패널, 로그, 로컬 폴더 작업, 애플리케이션 창 도우미 및 URL 분류자 유틸리티를 허용합니다. 의도적으로 DOM, 탐색, 리디렉션 및 브라우저 탭 도우미를 사용할 수 없거나 비활성인 것으로 처리합니다.

## 8. 로컬 파일 폴더

로컬 파일 폴더는 사용자 지정 규칙에 대해 사용자가 부여한 선택적 경계입니다. 규칙은 텍스트/CSV/JSON 읽기, 쓰기, 추가, 목록, 존재 테스트 및 JSON 작업을 요청할 수 있습니다. 경로는 항상 선택한 루트를 기준으로 합니다. 절대 경로, 순회 세그먼트, 숨겨진 경로 구성 요소, 지원되지 않는 확장 및 루트 외부 작업은 거부됩니다.

규칙에 더 이상 필요하지 않은 경우 폴더를 취소합니다. 규칙은 사용할 수 없는 권한과 실패한 localFileEvent 결과를 처리해야 합니다. 다시 시작한 후에도 선택한 폴더에 권한이 남아 있다고 가정해서는 안 됩니다.

## 9. 웹앱 브릿지

브리지는 호환되는 Vault 프로그램 간의 선택적 로컬 동기화입니다. 기본 데스크톱 앱은 로컬 허브를 호스팅할 수 있습니다. 클라이언트는 지원되는 로컬 주소에 연결됩니다.

연결 상태는 꺼짐, 연결 중, 연결 끊김, 연결됨/실행 중 및 오류입니다. 프로그램을 연결해도 모든 그룹이 병합되지는 않습니다. 사용자는 적격 일치 그룹을 명시적으로 연결해야 합니다.

그룹 링크의 경우:

1. 설정에서 기본 허브를 시작합니다.
2. 호환되는 다른 Vault 엔드포인트를 연결합니다.
3. 이름과 유형이 동일한 일치하는 고정 해제된 그룹을 만듭니다.
4. 그룹 브릿지 섹션에서 프로그램을 선택하고 그룹을 연결하세요.

연결된 그룹은 클러스터를 형성합니다. 지원되는 공통 정책 값, 사용량, 다시 알림 상태는 구성원이 연결되어 있는 동안 동기화할 수 있습니다. 연결을 끊으면 동기화가 일시 중지되고 로컬 그룹이 보존됩니다. 브라우저 전용 대상, 지원되지 않는 사용자 정의 작업 및 플랫폼별 필드는 전송이 보장되지 않습니다.

## 10. 가져오기, 내보내기, 재설정 및 감사

내보내기는 하나의 호환 가능한 그룹 표현을 저장합니다. 가져오기는 호환되는 그룹 데이터의 유효성을 검사/정규화하고 여전히 로컬 이름 고유성을 적용합니다. 그룹 삭제는 선택한 그룹 및 관련 상태를 제거합니다. 지우기는 확인 후 모든 그룹을 제거합니다. 기본값으로 재설정하면 전역 편집기 설정에 영향을 미칩니다. 먼저 보관해야 하는 것은 무엇이든 내보냅니다.

데스크탑 규칙을 사용하기 전에 다음을 수행하십시오.

1. 장치 제어가 부여되었는지 확인합니다.
2. 선택한 대상의 정규화된 ID를 확인합니다.
3. 활성화된 상태, 일정, 정지 상태 및 다시 알림 단계를 확인합니다.
4. 즉시, 시간 초과 및 카운트업 동작을 별도로 테스트합니다.
5. Custom 그룹의 경우 정확한 소스를 실행하고 등록된 각 앱 이벤트를 테스트합니다.
6. 로컬 폴더 실패와 성공적인 작업을 확인합니다.
7. 그룹이 연결된 경우 브리지 오프라인/연결 동작을 확인합니다.
