# Функциональная справка по настольному приложению Vault.

## Цель и границы

Это авторитетный справочник по интерфейсу настольного приложения Vault. Оно намеренно отделено от руководства по расширению браузера Vault.

Настольное приложение управляет **собственными приложениями и окнами приложений**. Расширение браузера управляет веб-сайтами, вкладками браузера и поддерживаемыми каналами веб-платформы. Они имеют одни и те же идеи — группы, расписания, таймеры, зависания, отсрочки, пользовательские правила и дополнительный мост — но у них разные возможности реализации.

Используйте этот документ для настройки, аудита, воспроизведения или поддержки поведения настольного приложения. Код является каноническим, если реализация и данное руководство различаются.

## 1. Что может и не может контролировать настольное приложение

Vault оценивает политику фокусировки для выбранных собственных приложений. Когда доступны встроенные возможности принудительного применения, он может применить текущий план к соответствующим целям приложения и сообщить результат защиты/статуса в пользовательский интерфейс хоста.

Он может:

- создавать, включать, отключать, изменять порядок, импортировать, экспортировать, замораживать, откладывать и удалять группы;
- целевые собственные приложения, выбранные с помощью средства выбора приложений;
- применить немедленный блок, временное разрешение или таймер только для подсчета;
- ограничить обычные группы будними днями и местными временными окнами;
- запускать пользовательские правила политики JavaScript для событий жизненного цикла приложения;
- показывать созданную правилами собственную информацию о статусе/панели через хост;
- управлять дополнительной локальной папкой для поддерживаемых запросов файлов по пользовательским правилам;
- присоединяйтесь к совместимым явно связанным группам через локальный мост Vault.

Он не может:

- действовать как расширение браузера, проверять DOM веб-сайта или манипулировать карточками новостей браузера;
- гарантировать, что операционная система позволит контролировать каждое приложение, процесс, окно или системную службу;
- превратить выбор приложений в удаленное администрирование, наблюдение за устройствами или межсетевой экран;
- заставить пользовательские помощники, доступные только в браузере, такие как DOM, навигация, перенаправление или управление вкладками, работать в собственной среде выполнения;
- автоматически синхронизировать каждую группу просто потому, что локальный мост работает.

## 2. Словарный запас

| Срок | Значение |
| --- | --- |
| Группа | Именованный объект политики фокуса. Имена групп должны быть уникальными в пределах текущей конечной точки Vault. |
| Цель | Собственный идентификатор приложения, выбранный для группы. |
| Группа приложений по умолчанию | Обычная группа, целью которой являются собственные приложения из средства выбора приложений. |
| Custom group | A group whose JavaScript rule reacts to application-policy events. |
| Матч | Текущее приоритетное/работающее приложение соответствует включенной и активной групповой цели или условию настраиваемого правила. |
| Активный | Включено, в рамках обычного расписания, а не активно отложено. |
| План исполнения | Итоговое решение о разрешении/защите/статусе собственного хоста после оценки применимых групп. |
| Заморозить | Защита от обычного изменения группы. |
| Отложить | Временное исключение из обычной групповой политики. |

## 3. Выбор целевого идентификатора и приложения

Выберите приложения с помощью средства выбора ***+** в группе приложений по умолчанию. В Vault хранится нормализованное удостоверение, а также отображаемое имя.

| Хозяин | Целевой идентификатор, используемый для сопоставления |
| --- | --- |
| macOS | Идентификатор пакета приложения, если он доступен. |
| Окна | Нормализованный путь к исполняемому файлу или имя процесса, предоставленное средством выбора приложения. |

Отображаемое имя предназначено для редактора. Нормализованное значение — это идентификатор, используемый собственным уровнем принудительного применения. Переименование приложения в пользовательском интерфейсе не меняет его идентичность. Цель также может содержать теги для использования политики настраиваемых правил.

Не вводите URL-адрес веб-сайта в поле целевого приложения и ожидайте принудительного применения встроенных приложений. Используйте группу «Сайт» расширения для блокировки веб-сайтов.

## 4. Жизненный цикл и приоритет группы

Новая группа включена по умолчанию. Список групп поддерживает выбор, включение/выключение, порядок перетаскивания, добавление, очистку, импорт, экспорт и удаление. Выбранная группа откроется в редакторе.

Обычные изменения полей сохраняются с помощью политики автосохранения редактора. Замороженная группа отключает обычные элементы управления редактированием. Пользовательский источник отличается: сохранение текста не делает его активным; **Запуск** — это операция, которая загружает текущий источник в среду выполнения политики.

Одному и тому же приложению могут соответствовать несколько групп. Vault оценивает групповую политику в сохраненном порядке и создает собственный план обеспечения соблюдения. Сохраняйте дублирование намеренно, особенно когда группы используют разные временные политики или пользовательские правила разрешают/защищают решения. Измените порядок групп, чтобы прояснить предполагаемый приоритет; не полагайтесь на то, что конфликтующая конфигурация будет разрешена особым, удобным для пользователя способом.

##5. Обычные группы приложений

### 5.1 Состояние группы

| Поле | Функциональный контракт |
| --- | --- |
| Имя | Непустой, обрезанный, уникальный без учета регистра в пределах этой конечной точки. |
| Включено | Группы инвалидов сохраняются, но не принимают участия в обычном правоприменении. |
| Цели | Один или несколько идентификаторов приложения, выбранных из средства выбора. |
| Поведение | Немедленная блокировка, блокировка после допуска или таймер/отсчет. |
| Расписание | Выбранные дни недели и дополнительные окна местного времени. |
| Заморозить | «Нет», «Заморожено», «Строго заморожено» или «Родительски заморожено». |
| Отложить | Политика временных исключений для каждой группы. |
| Резервное сообщение/сообщение о состоянии | Сообщение, которое может показывать собственный хост, когда он применяет ответ защиты/статуса. |

Пустая группа по умолчанию не имеет выбранного целевого приложения и, следовательно, не соответствует приложению просто по факту его существования.

### 5.2 Поведение блокировки

| Поведение | Результат |
| --- | --- |
| Немедленно заблокировать | Соответствующая активная цель приводит к немедленному принятию собственного решения о блокировании/защите. |
| Блокировать через несколько минут | Соответствующее использование засчитывается в счет группового пособия. Когда лимит исчерпан, группа выдает собственное решение о блокировке/защите до тех пор, пока период его использования не будет сброшен или другое состояние не сделает группу неактивной. |
| Таймер (обратный отсчёт, без блокировки) | Соответствующее использование измеряется и может отображаться, но сам по себе этот таймер никогда не создает блок. |

В новых группах используется 15-минутный интервал и 24-часовой интервал сброса, если это не изменено. Использование по времени принадлежит группе, поэтому все соответствующие цели разделяют эту групповую политику. Точный ответ на блокировку реализуется собственным хостом и ограничивается разрешениями операционной системы и поддерживаемым механизмом принудительного применения.

### 5.3 Расписания

Расписание распространяется на обычные группы. Пользовательская группа принимает собственные решения по времени в JavaScript.

Выберите любую комбинацию с понедельника по воскресенье. Каждое временное окно представляет собой одну строку местного времени:

```text
0900-1200
1330-1730
```

Точный принятый формат: ЧЧММ-ЧЧММ. Часы должны быть от 00 до 23, минуты от 00 до 59, а начало должно быть раньше окончания в тот же день. Окно включает начало и исключает конец. Окна, расположенные ближе к полуночи, не принимаются. Пустые окна означают весь выбранный день.

Обычная группа активна только тогда, когда:

1. он включен;
2. выбран текущий день недели;
3. местное время находится внутри настроенного окна или в группе нет окон;
4. дело не в активном режиме сна.

### 5.4 Отложить

Функция «Отложить» временно удаляет обычную группу из режима активного принудительного применения. Его фазы:

| Фаза | Результат |
| --- | --- |
| В ожидании | Запрос существует, но задержка активации не истекла; группа остается активной. |
| Активный | Группа временно неактивна на время ожидания. |
| Время восстановления | Откладывание завершилось, и группа снова активна, но новый отклад еще не доступен. |

| Настройка | Правило |
| --- | --- |
| Разрешить отложить | Если этот параметр выключен, группу нельзя нормально отложить. |
| Продолжительность повтора | Положительное количество минут. Значение по умолчанию для новой группы — 30 минут. |
| Задержка активации | Ноль или более минут до того, как режим повтора станет активным. |
| Время восстановления | От нуля до пяти минут после окончания активного повтора. |
| Подтверждения | Неотрицательное целое число необходимых подтверждающих взаимодействий. |

Активная отсрочка — это временное исключение из политики, а не удаление или разблокировка. Конфигурация группы остается неизменной.

### 5.5 Заморозка

Заморозка — это преднамеренный барьер модификации.

| Режим | Контракт |
| --- | --- |
| Замороженный | Обычные изменения и обычные изменения состояния остаются заблокированными до тех пор, пока поток подтверждения разблокировки продукта не завершится успешно. |
| Строгая замороженная | Группу нельзя разморозить до истечения срока ее строгой заморозки. Продолжительность положительна и ограничена 72 часами. |
| Родительский заморожен | Для действий по заморозке/разморозке требуется управление паролем-защитником. |

Выбор режима в редакторе сам по себе не замораживает группу; используйте действие замораживания, чтобы применить его. Группа, связанная мостом, также может заблокировать скоординированные элементы управления заморозкой, пока необходимый участник находится в автономном режиме.

## 6. Встроенное обеспечение и контроль устройств

Редактор может точно сохранить группу, даже если операционная система не предоставила возможность ее принудительного применения. Всегда проверяйте **Настройки → Контроль устройств** и текущий статус после изменения разрешений.

Собственный хост решает, какие действия возможны для текущей операционной системы, приложения, окна и состояния разрешений. Правило может быть правильно настроено, но не иметь видимого эффекта, если:

- Контроль устройства не предоставлен или отозван;
- группа отключена, отключена по расписанию или активно отложена;
- сфокусированный процесс не соответствует выбранной нормализованной цели;
- операционная система отклоняет действие для этой цели;
— зависимость моста отключена для действия, требующего скоординированного состояния.

Не рассматривайте всплывающее сообщение об успешном сохранении как доказательство того, что активное принудительное исполнение доступно. Проверьте выбранную цель, пока группа активна, и проверьте состояние хоста.

## 7. Пользовательские группы и собственные правила политики

Пользовательские группы выполняются во встроенной среде выполнения политики JavaScript. Это не пользовательские правила браузера. DOM браузера, вкладки, навигация, перенаправление URL-адресов и поведение управления лентами намеренно недоступны.

### 7.1 Жизненный цикл исходного кода

Use a function expression:

```js
(events, helpers) => {
  events.on("focusEvent", "shield-focus", (event, h) => {
    if (event.target?.id) event.setResult(-1);
  });
}
```

Run loads the source and its event registrations. Running again unloads the old source and resets its rule-owned handlers, timers, panels, persistence, and dynamic app blocklist. A source that does not evaluate to a function cannot be loaded.

### 7.2 Собственные встроенные события

| Событие | Значение |
| --- | --- |
| тикСобытие | Периодический тик хоста. Опция регистрации интервалаMs может ограничить скорость обработчика. |
| таймер закончился | Обратный отсчет, принадлежащий правилу, достигает нуля. |
| отложитьПресс | Пользователь нажимает кнопку «Начать повтор» для пользовательской группы. |
| панельСобытие | Используется элемент управления Custom Panel. |
| локальное событие | Запрошенное действие с локальной папкой завершено. |
| openAppEvent | Откроется отслеживаемое приложение. |
| закрытьAppEvent | Отслеживаемое приложение закрывается. |
| фокусСобытие | Приложение переднего плана изменится на приложение. |
| unfocusEvent | Приложение переднего плана меняется с приложения. |
| минимизироватьEvent / unminimizeEvent | Хост сообщает о поддерживаемом переходе к сворачиванию окна. |
| переключательAppEvent | Приложение переднего плана меняется от одного приложения к другому. |
| приложениеChangedEvent | Общее событие жизненного цикла/изменения приложения. |

Объект события содержит тип, идентификатор группы/идентификатор группы, имя группы, эквиваленты URL-адреса/имени хоста, время, данные и цель. Для собственного приложения target предоставляет идентификатор, вид, displayName, нормализованное значение и теги, когда цель фокуса соответствует настроенной цели.

Данные о событиях жизненного цикла приложения включают текущий идентификатор/имя приложения, имя группы, сериализованный снимок работающего приложения и значения, специфичные для события, такие как BundleId, previousAppId, currentAppId или причина изменения.

### 7.3 API событий и решения

Реестр обеспечивает включение/регистрацию, выключение/отмену регистрации, unregisterAll, countRegistered, getEvent и getEvents. Сначала выполняется более высокий приоритет; равный приоритет сохраняет порядок регистрации. В реестре имеется ограничение на количество обработчиков для каждой группы.

Объект события поддерживает:

| Метод | Результат |
| --- | --- |
| установитьРезультат(-1) | Создайте собственное решение по защите/блокированию. Строковый результат также становится собственным блоком, поскольку правила рабочего стола не имеют цели перенаправления браузера. |
| разрешить(причина) или setResult(1) | Выдайте решение о разрешении события. |
| setShieldMessage(сообщение) | Установите экран/сообщение о состоянии, обращенное к человеку, для собственного блока. |
| стопПропагация() | Остановите последующие обработчики текущего события. |
| заблокировать(appId), разблокировать(appId) | Добавить/удалить динамический блок собственного приложения. |
| закрыть(appId), открыть(appId) | Запросите поддерживаемое собственное действие закрытия/открытия. |
| сообщение(тип, данные) | Отправьте вложенное пользовательское событие в собственную среду выполнения. |

Среда выполнения приложения поддерживает таймеры, сохранение, панели, журналы, операции с локальными папками, помощники окон приложений и утилиты классификатора URL-адресов. Он намеренно рассматривает DOM, навигацию, перенаправление и помощники вкладок браузера как недоступные/инертные.

### 7.4 Родные помощники

| Помощник | Родное поведение |
| --- | --- |
| getLogHelper | Выдает решения журнала приложения/всплывающего окна/экрана. |

| getPersistenceHelper | Состояние JSON для каждой группы: получение, установка, удаление, наличие, ключи, записи, очистка, размер. |
| getStorageHelper | Постоянство плюс заполнители асинхронных запросов хоста; не предполагайте синхронного внешнего ответа. |
| getWindowHelper | Считывает текущие/работающие приложения и запрашивает действия по закрытию/открытию/блокировке/разблокировке приложений. |
| getPanelHelper | Создает проверенные собственные снимки панели, элементы управления, встроенные обработчики и реакции PanelEvent. |
| getLocalFolderHelper | Очереди допускали относительные операции с файлами .txt, .csv и .json в корневом каталоге, предоставленном пользователем. Завершением является localFileEvent. |
| getDomainHelper/getDomainUtility | Классификаторы URL-адресов и платформ для правил, которые также учитывают значения, подобные URL-адресам. |
| getPlatformHelper/платформа | Классификаторы URL остаются доступными; Собственные вызовы управления фидом/DOM инертны, поскольку у хоста рабочего стола нет DOM веб-сайта. |

Настраиваемые панели используют тот же словарь декларативного управления, что и среда выполнения браузера: текст, флажок, выбор, textInput, textarea, кнопка, раздел, таймер, NumberInput, диапазон, переключатель, радио, дата, время, цвет, PIN-код и очищенный HTML. Собственный хост решает, какая часть панели может отображаться на текущей платформе.

## 8. Локальная папка с файлами

Локальная папка с файлами — это необязательная, предоставляемая пользователем граница для пользовательских правил. Правила могут запрашивать чтение, запись, добавление текста/CSV/JSON, списки, тесты существования и операции JSON. Пути всегда указываются относительно выбранного корня. Абсолютные пути, сегменты обхода, скрытые компоненты пути, неподдерживаемые расширения и операции вне корня отклоняются.

Отзовите папку, если она больше не нужна правилу. Правило должно обрабатывать недоступные разрешения и неудачные результаты localFileEvent; он не должен предполагать, что выбранная папка остается авторизованной после перезапуска.

## 9. Мост через веб-приложение

Мост — это дополнительная локальная синхронизация между совместимыми программами Vault. В собственном настольном приложении может размещаться локальный концентратор; клиенты подключаются по поддерживаемому локальному адресу.

Состояния подключения: «Выключено», «Подключение», «Отключено», «Подключено/работает» и «Ошибка». Подключение программы не объединяет все группы. Пользователь должен явно связать подходящие соответствующие группы.

Для ссылки на группу:

1. Запустите родной хаб в Настройках.
2. Подключите другую совместимую конечную точку Vault.
3. Создайте соответствующие незамороженные группы с одинаковым именем и типом.
4. В разделе группового моста выбираем программу и подключаем группу.

Связанная группа образует кластер. Поддерживаемые общие значения политики, использование и состояние повтора могут синхронизироваться, пока участники подключены. Отключение приостанавливает синхронизацию и сохраняет локальные группы. Целевые объекты только для браузера, неподдерживаемые дополнительные действия и поля, специфичные для платформы, не могут быть перенесены.

## 10. Импорт, экспорт, сброс и аудит

Экспорт сохраняет одно совместимое представление группы. Импорт проверяет/нормализует совместимые данные группы и по-прежнему обеспечивает уникальность локального имени. Удалить группу удаляет выбранную группу и связанное с ней состояние. Очистить удаляет все группы после подтверждения. Сброс к настройкам по умолчанию влияет на глобальные настройки редактора; экспортируйте все, что необходимо сохранить в первую очередь.

Прежде чем полагаться на правило рабочего стола:

1. Убедитесь, что управление устройствами предоставлено.
2. Проверьте нормализованную идентичность выбранной цели.
3. Проверьте включенное состояние, расписание, состояние фиксации и фазу повтора.
4. Тестируйте немедленное, рассчитанное на время и подсчитывающее поведение отдельно.
5. Для пользовательской группы запустите точный источник и протестируйте каждое зарегистрированное событие приложения.
6. Проверьте сбои локальных папок, а также успешные операции.
7. Проверьте поведение моста в автономном режиме/подключении, если группа связана.
