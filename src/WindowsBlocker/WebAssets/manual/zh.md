# Vault 桌面应用程序功能参考

## 目的和边界

这是 Vault 桌面应用程序界面的权威参考。它有意与 Vault 浏览器扩展手册分开。

桌面应用程序管理**本机应用程序和应用程序窗口**。浏览器扩展管理网站、浏览器选项卡和支持的网络平台源。它们共享相同的想法——组、时间表、计时器、冻结、暂停、自定义规则和可选桥——但它们没有相同的执行表面。

使用本文档来配置、审核、重现或维护桌面应用程序行为。如果实现与本手册不同，则该代码是规范的。

## 1. 桌面应用可以控制什么，不能控制什么

Vault 评估所选本机应用程序的焦点策略。当其本机执行功能可用时，它可以将当前计划应用于匹配的应用程序目标并向主机 UI 报告屏蔽/状态结果。

它可以：

- 创建、启用、禁用、重新排序、导入、导出、冻结、暂停和删除组；
- 通过应用程序选择器选择的目标本机应用程序；
- 应用立即阻止、定时津贴或仅计数计时器；
- 将正常组限制为工作日和当地时间窗口；
- 针对应用程序生命周期事件运行自定义 JavaScript 策略规则；
- 通过主机显示规则创建的本机状态/面板信息；
- 管理支持的自定义规则文件请求的可选本地文件夹；
- 通过本地 Vault 桥加入兼容的显式链接组。

它不能：

- 充当浏览器扩展，检查网站 DOM，或操作浏览器提要卡；
- 保证操作系统允许控制每个应用程序、进程、窗口或系统服务；
- 将应用程序选择转变为远程管理、设备监控或防火墙；
- 使仅浏览器的自定义帮助程序（例如 DOM、导航、重定向或选项卡控件）在本机运行时工作；
- 仅因为本地网桥正在运行而自动同步每个组。

## 2. 词汇

|术语 |意义|
| --- | --- |
|集团|命名的焦点策略对象。组名称在当前 Vault 端点中必须是唯一的。 |
|目标|为组选择的本机应用程序标识。 |
|默认应用程序组|一个普通组，其目标是应用程序选择器中的本机应用程序。 |
| Custom group | A group whose JavaScript rule reacts to application-policy events. |
|比赛|当前前台/正在运行的应用程序与已启用且活动的组目标或自定义规则条件匹配。 |
|活跃|在正常计划内启用，并且未主动延后。 |
|执行计划|本机主机在评估适用组后做出的允许/屏蔽/状态决定。 |
|冻结|防止对组进行普通修改。 |
|打瞌睡 |正常组策略的临时例外。 |

## 3. 目标身份和应用选择器

通过默认应用程序组中的 **+** 选择器选择应用程序。 Vault 存储标准化身份和显示名称。

|主持人|用于匹配的目标身份 |
| --- | --- |
| macOS |应用程序包标识符（如果可用）。 |
|窗户|应用程序选择器提供的标准化可执行路径或进程名称。 |

显示名称用于编辑器。标准化值是本机执行层使用的身份。在 UI 中重命名应用程序不会更改其身份。目标还可以携带用于自定义规则策略使用的标签。

不要在应用程序目标字段中输入网站 URL，并期望本机应用程序强制执行。使用扩展程序的站点组进行网站阻止。

## 4. 组生命周期和优先级

默认情况下启用新组。组列表支持选择、启用/禁用、拖动排序、添加、清除、导入、导出和删除。所选组将在编辑器中打开。

正常的字段编辑通过编辑器的自动保存策略保存。冻结组会禁用普通编辑控件。自定义源则不同：保存文本并不使其处于活动状态； **运行** 是将当前源加载到策略运行时的操作。

多个组可以匹配同一个应用程序。 Vault 按存储顺序评估组策略并构建本机执行计划。有意保持重叠，特别是当团体使用不同的定时策略或自定义规则发布允许/屏蔽决策时。对组重新排序以明确预期的优先级；不要依赖以特定的用户友好方式解决冲突的配置。

## 5. 普通应用组

### 5.1 组状态

|领域|功能合同|
| --- | --- |
|名称 |此端点内非空、经过修剪、唯一（不区分大小写）。 |
|已启用 |残疾人群体被保留，但不参与正常执法。 |
|目标 |从选择器中选择的一个或多个应用程序标识。 |
|行为 |立即阻止、津贴后阻止或计时器/计数。 |
|日程 |选定的工作日和可选的当地时间窗口。 |
|冻结|无、冻结、严格冻结或家长冻结。 |
|打瞌睡 |每组临时例外政策。 |
|后备/状态消息 |本机主机在应用防护/状态响应时可以显示的消息。 |

空的默认组没有选定的应用程序目标，因此仅通过现有应用程序与应用程序不匹配。

### 5.2 阻止行为

|行为 |结果 |
| --- | --- |
|立即封锁 |匹配的活动目标会立即产生本地块/屏蔽决策。 |
|几分钟后阻止 |配套使用根据团体津贴累积。当限额用完时，该组会产生本机阻止/屏蔽决策，直到其使用期限重置或另一个状态使该组处于非活动状态。 |
|计时器（向上计数，无阻塞）|匹配的使用会被测量并可能被显示，但该计时器本身永远不会产生块。 |

除非有改变，新组使用 15 分钟的时间间隔和 24 小时的重置间隔。定时使用属于组，因此所有匹配的目标都共享该组策略。对块的确切响应由本机主机实现，并受到操作系统权限和支持的执行机制的约束。

### 5.3 时间表

时间表适用于普通团体。自定义组在 JavaScript 中做出自己的时间决定。

选择周一到周日的任意组合。每个时间窗口是当地时间的一行：

```text
0900-1200
1330-1730
```

确切接受的格式是 HHMM-HHMM。小时必须为 00 到 23，分钟为 00 到 59，并且当天的开始时间必须早于结束时间。窗口包括其开始并排除其结束。不接受跨午夜窗口。空窗口意味着整个选定的一天。

正常组仅在以下情况下才处于活动状态：

1. 已启用；
2.选择当前工作日；
3.本地时间在配置的窗口内，或者组没有窗口；
4. 没有处于主动小睡状态。

### 5.4 贪睡

暂停会暂时将正常组从主动执行中删除。其阶段是：

|相|结果 |
| --- | --- |
|待定 |请求存在但激活延迟尚未过去；该小组仍然活跃。 |
|活跃|该组在小睡时间内暂时不活动。 |
|冷却时间|暂停已结束，该组再次活跃，但新的暂停尚未可用。 |

|设置|规则|
| --- | --- |
|允许小睡 |关闭时，该组不能正常休眠。 |
|贪睡持续时间 |正分钟数。新组的默认时间为 30 分钟。 |
|激活延迟|贪睡开始前零分钟或更多分钟。 |
|冷却时间|主动小睡结束后零到五分钟。 |
|确认|所需确认交互的非负整数。 |

主动暂停是临时策略例外，而不是删除或解冻。组配置保持不变。

### 5.5 冻结

冻结是一种刻意的修改障碍。

|模式|合同|
| --- | --- |
|冷冻|普通编辑和普通状态更改保持锁定，直到产品的解冻确认流程成功。 |
|严格冷冻|在严格冻结期限结束之前，无法解冻该组。持续时间为正且限制为 72 小时。 |
|父母冷冻|冻结/解冻操作需要监护人密码管理。 |

在编辑器中选择模式不会自行冻结组；使用冻结操作来应用它。当所需成员离线时，桥接组还可以锁定协调的冻结控制。

## 6. 本机执行和设备控制

即使操作系统没有授予强制执行组的能力，编辑器也可以准确地保存组。更改权限后，请务必检查 **设置 → 设备控制** 和实时本机状态。

本机主机决定当前操作系统、应用程序、窗口和权限状态可以执行哪些操作。在以下情况下，可以正确配置规则但没有可见效果：

- 设备控制未授予或已被撤销；
- 该组已禁用、已计划退出或已主动延后；
- 重点流程与选定的标准化目标不匹配；
- 操作系统拒绝针对该目标的操作；
- 对于需要协调状态的操作，桥依赖项处于脱机状态。

不要将成功的 save toast 视为可以主动执行的证据。在组处于活动状态时测试所选目标并检查主机状态。

## 7. 自定义组和本机策略规则

自定义组在本机 JavaScript 策略运行时运行。它们不是浏览器自定义规则。浏览器 DOM、选项卡、导航、URL 重定向和提要控制行为故意不可用。

### 7.1 源生命周期

Use a function expression:

```js
(events, helpers) => {
  events.on("focusEvent", "shield-focus", (event, h) => {
    if (event.target?.id) event.setResult(-1);
  });
}
```

Run loads the source and its event registrations. Running again unloads the old source and resets its rule-owned handlers, timers, panels, persistence, and dynamic app blocklist. A source that does not evaluate to a function cannot be loaded.

### 7.2 本机内置事件

|活动 |意义|
| --- | --- |
|勾选事件 |周期性主机滴答声。 IntervalMs 注册选项可以对处理程序进行速率限制。 |
|计时器已结束 |规则拥有的倒计时为零。 |
|打盹按 |用户按自定义组的开始暂停。 |
|面板事件 |使用自定义面板控件。 |
|本地文件事件 |请求的本地文件夹操作完成。 |
|打开应用程序事件 |跟踪的应用程序将打开。 |
|关闭应用程序事件 |跟踪的应用程序关闭。 |
|焦点事件 |前台应用程序更改为应用程序。 |
|取消焦点事件 |前台应用程序不再是一个应用程序。 |
|最小化事件/取消最小化事件 |主机报告支持的窗口最小化转换。 |
|切换应用程序事件 |前台应用程序从一个应用程序更改为另一个应用程序。 |
|应用程序更改事件 |一般应用程序生命周期/更改事件。 |

事件对象包含类型、groupId/groupID、groupName、URL/主机名等效项、时间、数据和目标。对于本机应用程序，当焦点目标与配置的目标匹配时，目标会公开 id、kind、displayName、标准化值和标签。

应用程序生命周期事件数据包括当前应用程序 ID/名称、组名称、序列化的运行应用程序快照以及特定于事件的值，例如bundleId、previousAppId、currentAppId 或更改原因。

### 7.3 事件 API 和决策

注册表提供 on/register、off/unregister、unregisterAll、countRegistered、getEvent 和 getEvents。优先级高的先运行；同等优先级保留注册顺序。注册表具有每组处理程序上限。

事件对象支持：

|方法|结果 |
| --- | --- |
|设置结果(-1) |产生原生屏蔽/格挡决策。字符串结果也会成为本机块，因为桌面规则没有浏览器重定向目标。 |
|允许（原因）或 setResult(1) |为事件生成允许决定。 |
| setShieldMessage(消息) |设置本机块的面向人的屏蔽/状态消息。 |
|停止传播（）|停止当前事件的后续处理程序。 |
|阻止（appId），解除阻止（appId）|添加/删除动态本机应用程序块。 |
|关闭（appId），打开（appId）|请求支持的本机关闭/打开操作。 |
|发布（类型，数据）|在本机运行时中调度嵌套的自定义事件。 |

应用程序运行时允许计时器、持久性、面板、日志、本地文件夹操作、应用程序窗口帮助程序和 URL 分类器实用程序。它故意将 DOM、导航、重定向和浏览器选项卡帮助程序视为不可用/惰性。

### 7.4 本机助手

|帮手|本机行为 |
| --- | --- |
|获取日志助手 |发出应用程序/弹出窗口/屏幕日志决策。 |
|获取定时器助手 |创建具有边界、步骤、范围/域谓词、暂停/恢复、状态检查和timerEnded 转换的前向/后向计时器。定时器本身不屏蔽。 |
|获取持久性助手 |每组 JSON 状态：获取、设置、删除、具有、键、条目、清除、大小。 |
|获取存储助手 |持久性加上主机异步请求占位符；不要假设同步外部响应。 |
|获取窗口助手 |读取当前/正在运行的应用程序并请求关闭/打开/阻止/取消阻止应用程序操作。 |
|获取面板助手 |创建经过验证的本机面板快照、控件、内联处理程序和 panelEvent 反应。 |
|获取本地文件夹助手 |队列允许在用户授予的根目录下进行相对的 .txt、.csv 和 .json 操作。完成是localFileEvent。 |
| getDomainHelper / getDomainUtility | getDomainHelper / getDomainUtility |规则的 URL 和平台分类器也推理类似 URL 的值。 |
| getPlatformHelper / 平台 | URL 分类器仍然可用；本机 feed/DOM 控制调用是惰性的，因为桌面主机没有网站 DOM。 |

自定义面板使用与浏览器运行时相同的声明性控制词汇：text、checkbox、select、textInput、textarea、button、section、timer、numberInput、range、toggle、radio、date、time、color、pin 和 sanitized html。本机主机决定面板可以在当前平台上显示多少内容。

## 8. 本地文件夹

本地文件夹是可选的、用户授予的自定义规则边界。规则可以请求文本/CSV/JSON 读取、写入、追加、列表、存在测试和 JSON 操作。路径始终相对于选定的根。绝对路径、遍历段、隐藏路径组件、不支持的扩展以及根之外的操作都会被拒绝。

当规则不再需要该文件夹时，撤销该文件夹。规则必须处理不可用的权限和失败的 localFileEvent 结果；它不能假定所选文件夹在重新启动后仍保持授权。

## 9. Web 应用程序桥

该桥是兼容 Vault 程序之间可选的本地同步。本机桌面应用程序可以托管本地集线器；客户端连接到支持的本地地址。

连接状态包括“关闭”、“正在连接”、“已断开连接”、“已连接/正在运行”和“错误”。连接程序不会合并所有组。用户必须明确链接符合条件的匹配组。

对于群组链接：

1. 在“设置”中启动本机集线器。
2. 连接其他兼容的 Vault 端点。
3. 创建具有相同名称和类型的匹配的解冻组。
4. 在组桥接部分中，选择程序并连接组。

链接的基团形成簇。支持的通用策略值、使用情况和暂停状态可以在成员连接时同步。断开连接会暂停同步并保留本地组。不保证仅浏览器目标、不受支持的自定义操作和特定于平台的字段能够传输。

## 10. 导入、导出、重置和审核

导出保存一种兼容的组表示。导入验证/标准化兼容组数据，并仍然强制本地名称唯一性。删除组将删除选定的组及其关联状态。确认后清除会删除所有组。重置为默认值会影响全局编辑器设置；导出必须首先保留的任何内容。

在依赖桌面规则之前：

1. 验证设备控制是否已授予。
2. 验证所选目标的规范化身份。
3. 验证启用状态、计划、冻结状态和暂停阶段。
4. 分别测试立即、定时和计数行为。
5. 对于自定义组，运行确切的源并测试每个注册的应用程序事件。
6. 验证本地文件夹失败以及操作是否成功。
7. 如果该组已链接，请验证网桥离线/连接行为。

## 11. 平台特定注释

核心策略概念是共享的，但本机执行是特定于主机的：

| macOS |窗户|
| --- | --- |
|目标通常解析为应用程序包标识符。设备控制和当前 macOS 权限状态门强制执行。 |目标通常解析为规范化的可执行路径或进程名称。 Windows 执行层决定可以管理哪些当前窗口/进程。 |

此桌面参考故意不描述网站阻止列表、提要选择器、YouTube 创建者分类、浏览器重定向或浏览器选项卡操作。这些属于 Vault 扩展手册。
