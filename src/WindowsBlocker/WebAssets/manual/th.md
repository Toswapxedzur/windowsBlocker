# Custom Web Blocker — คู่มือการใช้งาน

นี่คือคู่มืออ้างอิงฉบับเต็มสำหรับส่วนขยาย เริ่มต้นด้วยขั้นตอนการทำงานที่ง่ายที่สุดและแพร่หลายที่สุด และค่อยๆ ย้ายไปยังหัวข้อขั้นสูง เช่น กฎการบล็อกที่ขับเคลื่อนด้วยเหตุการณ์แบบกำหนดเอง และ API ตัวช่วย

หากคุณเป็นผู้ใช้ใหม่ เพียงอ่าน **เริ่มต้นอย่างรวดเร็ว** และ **ภาพรวมกลุ่มบล็อก** ทุกอย่างที่อยู่ด้านล่างส่วนเหล่านั้นเป็นทางเลือก ขึ้นอยู่กับสิ่งที่คุณต้องการทำ

---

## 1. ส่วนขยายนี้ทำอะไร

Custom Web Blocker ช่วยให้คุณสามารถบล็อกเว็บไซต์และสิ่งรบกวนออนไลน์ตามกฎที่คุณกำหนดได้ คุณสามารถ:

- บล็อกไซต์ทันทีด้วยการบล็อกเครือข่ายดั้งเดิมของเบราว์เซอร์ (บล็อกประเภทเดียวกับที่สร้าง `ERR_BLOCKED_BY_CLIENT`)
- ให้เวลาตัวเองในไซต์หนึ่งๆ สักนาทีต่อวัน จากนั้นจึงบล็อกไซต์เมื่อคุณใช้เกินขีดจำกัดนั้น
- บล็อกเนื้อหาบางประเภทบน ยูทูบ, ติ๊กต็อก, เฟซบุ๊ก, อินสตาแกรม, ทวิช และ เรดดิต (ไม่ใช่ทั้งไซต์)
- ซ่อนเนื้อหาที่ถูกบล็อกจากฟีดบนแพลตฟอร์มที่รองรับ แทนที่จะบล็อกเพียงหน้าเดียว
- กำหนดเวลาที่กฎใช้งานตามวันในสัปดาห์และตามกรอบเวลา `HHMM-HHMM`
- หยุดกฎเพื่อไม่ให้คุณเปลี่ยนแปลงได้ง่าย การแช่แข็งแบบเข้มงวดจะล็อคไว้ตามจำนวนชั่วโมงที่ระบุ และต้องมีขั้นตอนการยืนยัน 20 ขั้นตอนในการเลิกทำ
- เลื่อนกฎชั่วคราว แต่หลังจากเขียนเหตุผลยาวพอแล้วเท่านั้น
- เขียนกฎที่กำหนดเอง **ที่ขับเคลื่อนด้วยเหตุการณ์** ใน ภาษาสคริปต์ พร้อมด้วยตัวช่วยสำหรับตัวจับเวลาเดินหน้า/ถอยหลัง พื้นที่เก็บข้อมูลถาวรต่อกลุ่ม เจตนา DOM ต่อแพลตฟอร์ม (ซ่อนปุ่มนำทาง ซ่อนการ์ดฟีดตามเพรดิเคต ตั้งค่าตัวจับเวลาต่อส่วนย่อย) ยูทิลิตี URL และการบันทึกที่มีโครงสร้าง
- เลือกจากไลบรารีเทมเพลตสำเร็จรูปในตัวที่มีมากกว่า 50 แบบ (ตัวจับเวลา ตารางเวลา การซ่อนฟีด เซสชันโฟกัส การเปลี่ยนเส้นทาง การกระตุ้นเตือน การคงอยู่ การปรับแต่ง DOM ตัวช่วยแก้ไขข้อบกพร่อง)
- ใช้ส่วนขยายในกว่า 20 ภาษา

ส่วนขยายนี้เป็นส่วนขยาย โครม Manifest V3 ที่มีหน้าตัวแก้ไขหนึ่งหน้า (ป๊อปอัป) พนักงานบริการพื้นหลังหนึ่งคน แซนด์บ็อกซ์นอกหน้าจอหนึ่งตัวที่โฮสต์โค้ดกฎที่กำหนดเอง และสคริปต์เนื้อหาหนึ่งตัวที่ทำงานในทุกหน้า กฎที่กำหนดเองจะแสดงอยู่ในแซนด์บ็อกซ์นอกหน้าจอ โดยจะโหลดหนึ่งครั้งต่อการคลิกเรียกใช้และคงการลงทะเบียนไว้จนกว่ากฎจะถูกปิดใช้งานหรือลบ

## 2. ทัวร์ชม UI

เมื่อคุณคลิกไอคอนส่วนขยาย ตัวแก้ไขจะเปิดขึ้นเป็นหน้าเว็บแบบเต็ม (ไม่ใช่ป๊อปอัปเล็กๆ) เพจมีพื้นที่เหล่านี้:

- **แถบด้านบน**
  - ปุ่ม **คู่มือการใช้งาน** (เอกสารนี้)
  - เครื่องมือเลือก **ภาษา**
  - **การตั้งค่า** เกียร์ (สลับขั้นสูง รวมถึง **โหมดแก้ไขข้อบกพร่อง**)
- **แผงด้านซ้าย — กลุ่มบล็อก**
  - รายชื่อกลุ่มบล็อกของคุณ การ์ดแต่ละใบจะแสดงชื่อกลุ่ม บรรทัดสรุปสั้นๆ และช่องทำเครื่องหมายเปิด/ปิด
  - ปุ่ม **เพิ่ม** สร้างกลุ่มใหม่ เมนูแบบเลื่อนลงถัดจากนั้นจะเลือกประเภท
  - **ลบทั้งหมด** จะลบทุกกลุ่ม พร้อมการยืนยันเพิ่มเติมหากมีกลุ่มใดถูกตรึงไว้
  - คุณสามารถลากที่จับ `::` บนการ์ดขึ้นหรือลงเพื่อจัดลำดับกลุ่มใหม่ได้
  - คุณสามารถลากตัวแยกแนวตั้งเพื่อปรับขนาดแผงนี้ได้
- **แผงด้านขวา — ตัวแก้ไข**
  - แก้ไขกลุ่มที่เลือกในปัจจุบัน: ชื่อ, พฤติกรรมการบล็อก, รายการบล็อก, ตัวกรองเฉพาะประเภท, กำหนดการ, หยุด, เลื่อน
  - การเปลี่ยนแปลงทั้งหมดจะบันทึกเสี้ยววินาทีโดยอัตโนมัติหลังจากที่คุณหยุดพิมพ์หรือโต้ตอบ
  - สำหรับกลุ่ม **กำหนดเอง** ตัวแก้ไขยังแสดงเบราว์เซอร์ **เทมเพลต** ปุ่ม **เรียกใช้** และแผง **บันทึก** (เปลี่ยนชื่อจาก *บันทึกกิจกรรม* ในเวอร์ชัน 1.1)
- **ขนมปังปิ้ง** (ป๊อปอัปตรงกลางที่จางลง) — แสดงข้อความสถานะ เช่น "การเปลี่ยนแปลงที่บันทึกไว้" หรือข้อผิดพลาดในการป้อนข้อมูล
- **โอเวอร์เลย์ในเพจ** — ในขณะที่แท็บมีตัวจับเวลาหรือบล็อกที่ทำงานอยู่ โอเวอร์เลย์จะปรากฏที่มุมซ้ายบนโดยแสดงข้อจำกัดทั้งหมดที่ส่งผลต่อแท็บในรูปแบบ `hh:mm:ss` (หรือ `mm:ss`) ข้อจำกัดหลายรายการซ้อนกันบนหลายบรรทัด การนับถอยหลังเริ่มต้นของกลุ่มบล็อกและตัวจับเวลากฎที่กำหนดเองแชร์การซ้อนทับนี้

## 3. เริ่มต้นอย่างรวดเร็ว1. คลิกไอคอนส่วนขยาย ตัวแก้ไขจะเปิดขึ้นเป็นหน้าเต็ม
2. ในแผง **บล็อกกลุ่ม** ให้เลือกประเภทกลุ่มจากเมนูแบบเลื่อนลง:
   - `Default`, `ยูทูบ`, `ติ๊กต็อก`, `เฟซบุ๊ก`, `อินสตาแกรม`, `ทวิช`, `เรดดิต` หรือ `Custom`
3. คลิก **เพิ่ม** กลุ่มใหม่จะปรากฏขึ้น และผู้แก้ไขจะเปิดขึ้นมา
4. ตั้งชื่อให้มัน.
5. กรอกข้อมูลในช่องเฉพาะประเภท (สำหรับ `Default` นั่นหมายถึงรายการ **เว็บไซต์ที่ถูกบล็อก**)
6. ตรวจสอบให้แน่ใจว่าช่องทำเครื่องหมายของกลุ่มในแผงด้านซ้ายเปิดอยู่
7. เยี่ยมชมหนึ่งในไซต์ที่ระบุไว้ การบล็อกควรมีผลทันที

นั่นคือเส้นทางแห่งความสุขทั้งหมด ส่วนที่เหลือของคู่มือนี้เป็นเพียงตัวเลือกนอกเหนือจากนี้

> เมื่อคุณกด **เรียกใช้** บนกลุ่มแบบกำหนดเอง กฎใหม่จะแนบกับกิจกรรมของเพจ **ในอนาคต** แท็บที่เปิดอยู่แล้วจะเรียกใช้กฎก่อนหน้าต่อไปจนกว่าคุณจะโหลดซ้ำ ป๊อปอัพจะแสดงการเตือนถึงเอฟเฟกต์นั้นหลังจากการวิ่งสำเร็จทุกครั้ง

## 4. ภาพรวมกลุ่มบล็อก

ทุกอย่างในส่วนขยายนี้จัดเป็น **กลุ่มบล็อก** กลุ่มบล็อกคือชุดกฎเดียว:

- มีชื่อ ประเภท และสถานะเปิด/ปิดใช้งาน
- มีพฤติกรรมการบล็อก (ทันที หลังจากผ่านไปหลายนาที หรือนับถอยหลังคงที่)
- มีกำหนดการเพิ่มเติม (วัน + กรอบเวลา) และการควบคุมการแช่แข็ง/เลื่อนซ้ำ
- มีช่องเพิ่มเติม เช่น รายชื่อเว็บไซต์ ตัวกรองผู้สร้าง ยูทูบ ชื่อ subreddit หรือกฎ ภาษาสคริปต์ ที่ขับเคลื่อนด้วยเหตุการณ์ ทั้งนี้ขึ้นอยู่กับประเภท

คุณสามารถมีกลุ่มจำนวนเท่าใดก็ได้ อาจใช้หลายกลุ่มในหน้าเดียวกัน ในกรณีนั้นกฎ **เข้มงวดที่สุด** จะชนะ:

- "บล็อกทันที" เต้น "บล็อกหลังจากผ่านไประยะหนึ่ง"
- กลุ่มที่มีเวลาเหลือน้อยกว่าจะชนะกลุ่มที่มีเวลาเหลือมากกว่า

ดังนั้นการเพิ่มกลุ่มจะทำให้การบล็อกเพจเร็วขึ้นเท่านั้น

**ลำดับการประเมินจะเรียงลำดับจากล่างขึ้นบน** เมื่อส่วนขยายวนซ้ำกลุ่มบล็อกของคุณ ส่วนขยายจะเริ่มต้นด้วยกลุ่มที่อยู่ด้านล่างสุดของรายการและไต่ระดับขึ้นไป กลุ่มที่อยู่ด้านบนสุดของรายการจะได้รับการประเมินเป็นอันดับสุดท้ายและได้รับ "คำสุดท้าย" ตัวอย่างเช่น หากกลุ่มที่อยู่ด้านล่างเรียก `helpers.getPlatformHelper().youtube().hideShortButton()` และกลุ่มที่อยู่ด้านบนเรียก `showShortButton()` ปุ่มจะยังคงมองเห็นได้ ลากที่จับ `::` บนการ์ดเพื่อเปลี่ยนลำดับนี้

## 5. ประเภทกลุ่ม

### 5.1 `Default` — บล็อกเว็บไซต์ธรรมดา

สำหรับการบล็อกโดเมนเฉพาะ (กรณีการใช้งานทั่วไป)

- **เว็บไซต์ที่ถูกบล็อก**: หนึ่งไซต์ต่อบรรทัด ทั้ง `facebook.com` และ `https://www.facebook.com/somepage` ทำงาน ส่วนขยายจะแยกและทำให้ชื่อโฮสต์เป็นมาตรฐาน
- กฎของไซต์ใช้กับชื่อโฮสต์นั้นและโดเมนย่อยทั้งหมด
- ประเภทกลุ่มนี้ใช้การบล็อกเครือข่ายดั้งเดิมของ โครม คล้ายกับ `ERR_BLOCKED_BY_CLIENT` นั่นหมายความว่าการนำทางไปยัง URL ที่ถูกบล็อกจะหยุดลงก่อนที่เพจจะโหลดด้วยซ้ำ

### 5.2 `ยูทูบ` — บล็อก ยูทูบ และไซต์วิดีโอที่คล้ายกัน

เพิ่มส่วน **ตัวกรอง** ให้กับตัวแก้ไข:

- **ประเภทเนื้อหา**:
  - `Apply to all ยูทูบ pages` — ทุกหน้า ยูทูบ มีความสำคัญ
  - `Apply to Shorts` — นับเฉพาะหน้าวิดีโอสั้นเท่านั้น
  - `Apply to long videos` — เฉพาะ `/watch`, `/live/`, `/embed/` เป็นต้น
  - `Apply to ยูทูบ posts` — โพสต์ชุมชน (`/post/...` แท็บชุมชนช่อง/โพสต์)
- **ตัวกรองผู้เขียน**:
  - `Do not filter by author` — ตัวตนของผู้เขียนไม่สำคัญ
  - `Apply to certain authors` — เฉพาะผู้เขียนที่อยู่ในรายชื่อเท่านั้นที่เรียกกลุ่มนี้
  - `Apply to all except certain authors` — ผู้เขียนที่อยู่ในรายชื่อจะได้รับการยกเว้น
- **ผู้เขียน**: หนึ่งผู้เขียนต่อบรรทัด ยอมรับ `@handle`, URL แบบเต็ม, `/channel/UC...`, `/c/...`, `/user/...`
- **ซ่อนรายการที่ถูกบล็อกในฟีด ยูทูบ**: ขณะที่กลุ่มนี้กำลังบล็อกอยู่ การ์ดที่ตรงกันในฟีด ยูทูบ จะถูกซ่อนไว้ เมื่อบล็อกไม่ทำงาน บล็อกจะกลับมาในการรีเฟรชครั้งถัดไป

สำหรับประเภทเนื้อหาสั้นและโพสต์ เมื่อไม่ได้ตั้งค่าตัวกรองผู้เขียนและกลุ่มกำลังบล็อกอยู่ ส่วนขยายจะซ่อนรายการนำทางที่เกี่ยวข้อง (รายการในแถบด้านข้าง Shorts แท็บช่องชุมชน/โพสต์) และชั้นวางที่ตรงกัน เช่น "โพสต์ ยูทูบ ล่าสุด"

การตรวจจับแบบสั้นและแบบยาวจะขยายไปยังไซต์วิดีโออื่นๆ เช่น ติ๊กต็อก, Vimeo, คลิป ทวิช/VOD และ Dailymotion เมื่อสามารถตรวจพบรูปแบบเพจได้

### 5.3 `ติ๊กต็อก` — บล็อกเนื้อหา ติ๊กต็อก

การ์ดตัวแก้ไขแบบเดียวกับตัวแก้ไขวิดีโอแพลตฟอร์ม แต่มีป้ายกำกับเฉพาะของ ติ๊กต็อก:- ประเภทเนื้อหา: วิดีโอสั้น วิดีโอ หน้าโปรไฟล์
- ผู้เขียน: ติ๊กต็อก จัดการ (`@handle`) หรือ URL โปรไฟล์
- การซ่อนฟีดจะซ่อนการ์ดที่ตรงกันบนหน้า ติ๊กต็อก ในขณะที่กลุ่มใช้งานอยู่

### 5.4 `เฟซบุ๊ก` — บล็อกเนื้อหา เฟซบุ๊ก

- ประเภทเนื้อหา: คลิปม้วน วิดีโอ โพสต์
- ผู้เขียน: ชื่อหน้า (`page.name`), URL โปรไฟล์ หรือแบบฟอร์ม `profile.php?id=...` (รหัสตัวเลขจะคงไว้เป็น `id:<number>`)
- การซ่อนฟีดซ่อนการ์ดฟีดที่ตรงกันบน เฟซบุ๊ก

### 5.5 `อินสตาแกรม` — บล็อกเนื้อหา อินสตาแกรม

- ประเภทเนื้อหา: คลิปม้วน วิดีโอ โพสต์
- ผู้เขียน: ตัวจัดการ อินสตาแกรม หรือ URL โปรไฟล์
- เส้นทางที่สงวนไว้ เช่น `/reel/`, `/p/`, `/tv/`, `/explore/` จะไม่ถือเป็นผู้เขียน
- ฟีดซ่อนซ่อนการ์ดที่ตรงกันบน อินสตาแกรม

### 5.6 `ทวิช` — บล็อกเนื้อหา ทวิช

- ประเภทเนื้อหา: คลิป สตรีม/VOD หน้าช่อง
- ผู้เขียน: ชื่อช่องหรือ URL ของช่อง
- เส้นทางที่สงวนไว้ เช่น `/directory`, `/videos`, `/settings` ฯลฯ จะไม่ถือเป็นชื่อช่อง
- การซ่อนฟีดซ่อนการ์ดที่ตรงกันบน ทวิช

### 5.7 `เรดดิต` — บล็อก เรดดิต หรือ subreddits เฉพาะ

- **Subreddits**: หนึ่ง subreddit ต่อบรรทัด รายการว่างหมายความว่ากลุ่มมีผลกับ เรดดิต ทั้งหมด ยอมรับทั้ง `productivity` และ `r/productivity`

### 5.8 `Custom` — บล็อกโดย ภาษาสคริปต์ ที่ขับเคลื่อนด้วยเหตุการณ์

คุณเขียนฟังก์ชัน ภาษาสคริปต์ ที่ **ลงทะเบียนตัวจัดการ** สำหรับเหตุการณ์ต่างๆ เช่น การเปิดเพจ การเปลี่ยน URL ฮาร์ทบีทของหน้า การสิ้นสุดตัวจับเวลา และเหตุการณ์ที่คุณกำหนดเอง ฟังก์ชันทำงานหนึ่งครั้งต่อการคลิกเรียกใช้; ตัวจัดการที่ลงทะเบียนไว้จะยังคงทำงานอยู่ในการนำทางจนกว่าคุณจะกดเรียกใช้อีกครั้ง ปิดการใช้งานกลุ่ม หรือลบออก

กลุ่ม `Custom` ไม่แสดง: พฤติกรรมการบล็อก ไซต์ที่ถูกบล็อก นาทีที่อนุญาต ช่วงเวลารีเซ็ต วันที่กำหนด หรือกรอบเวลา พวกเขาเก็บตัวแก้ไข **กฎการบล็อก** พร้อมด้วยการควบคุมการหยุด/เลื่อนแบบมาตรฐาน นอกจากนี้ยังมีปุ่ม **เทมเพลต** ที่เปิดเบราว์เซอร์ที่ตั้งค่าไว้ล่วงหน้าพร้อมกฎเริ่มต้นแบบกำหนดพารามิเตอร์ การใช้การตั้งค่าล่วงหน้าจะแทนที่กฎปัจจุบันหลังจากการยืนยัน

ดู **ส่วนที่ 11** สำหรับการอ้างอิงกฎที่กำหนดเองแบบเต็มและ API ตัวช่วย

## 6. พฤติกรรมการบล็อก

สำหรับประเภทกลุ่มส่วนใหญ่ คุณจะเลือกหนึ่งในสามโหมด

### 6.1 บล็อกทันที

กฎจะทำงานทุกครั้งที่เปิดกลุ่ม ตามกำหนดเวลา และ (สำหรับกลุ่มแพลตฟอร์ม) เพจจะตรงกัน

สำหรับกลุ่ม `Default` จะใช้การบล็อกดั้งเดิมของ โครม สำหรับกลุ่มแพลตฟอร์ม จะใช้ตรรกะการซ้อนทับ/ออกในเพจ

### 6.2 บล็อกหลังจากผ่านไปหลายนาที

นี่คืองบประมาณการใช้งาน

- **นาทีที่อนุญาตก่อนบล็อก** (ทศนิยม): คุณเผื่อเวลาไว้กี่นาทีต่อช่วง ตัวอย่าง: `15`, `0.5`, `90`
- **ช่วงเวลารีเซ็ตตัวจับเวลา (ชั่วโมง)** (ทศนิยม): ความถี่ในการรีเซ็ตงบประมาณ ตัวอย่าง: `24` สำหรับรายวัน, `1` สำหรับรายชั่วโมง, `0.25` ทุกๆ 15 นาที

ในขณะที่คุณมีเวลาเหลือ หน้าเว็บก็ทำงานได้ตามปกติและแสดงการซ้อนทับตัวจับเวลา เมื่องบประมาณถึงศูนย์ เพจจะถูกบล็อกในช่วงเวลาที่เหลือและโอเวอร์เลย์จะแสดง `0:00` จากนั้นแท็บจะพยายามออก

การขยายเวลาเป็นแบบต่อกลุ่ม ต่อช่วง:

- แต่ละกลุ่มมีงบประมาณของตัวเอง
- เวลาที่ใช้ในเพจใดๆ ที่ตรงกับกลุ่มจะถูกนับรวมในงบประมาณของกลุ่มนั้น
- หลายแท็บในกลุ่มเดียวกันใช้งบประมาณร่วมกัน ตัวจับเวลาของพวกเขายังคงซิงโครไนซ์อยู่ การสลับไปที่แท็บอื่นยังบังคับให้รีเฟรชด้วย ดังนั้นมันจึงแสดงเวลาที่ใช้ร่วมกันในปัจจุบันทันที

หากมีกลุ่มที่จำกัดเวลาหลายกลุ่มนำไปใช้กับหน้าเดียวกัน กลุ่มที่เข้มงวดที่สุดจะเป็นผู้ชนะ

### 6.3 ตัวจับเวลา (นับถอยหลังแล้วบล็อก)

โหมดนี้จะแสดงตัวจับเวลาถอยหลังและบล็อกเมื่อถึง `0:00`

- **ช่วงเวลารีเซ็ตตัวจับเวลา (ชั่วโมง)** (ทศนิยม): ทั้งความยาวตัวจับเวลาและความถี่ในการรีเซ็ต ตัวอย่าง: `24` สำหรับรายวัน, `1` สำหรับรายชั่วโมง, `0.25` ทุกๆ 15 นาที

ต่างจาก **บล็อกหลังจากผ่านไปหลายนาที** โหมดนี้ **ไม่** มีฟิลด์ "นาทีที่อนุญาตก่อนบล็อก" แยกต่างหาก ตัวจับเวลาจะเริ่มต้นที่ช่วงเวลารีเซ็ต นับถอยหลังในขณะที่หน้าที่ตรงกันเปิดอยู่ จากนั้นบล็อกจนกว่าจะรีเซ็ตครั้งถัดไปการนับถอยหลังแบบกลุ่มเริ่มต้นและตัวจับเวลาแบบกลุ่มแบบกำหนดเอง (ดู **ส่วนที่ 11.3.1**) ทั้งคู่ **เลื่อนไปข้างหน้าเมื่อมองเห็นแท็บเท่านั้น** การสลับแท็บ ย่อหน้าต่าง หรือการล็อคหน้าจอจะหยุดการนับถอยหลังชั่วคราวโดยอัตโนมัติ

## 7. กำหนดการ

ในการ์ด **กำหนดการ** คุณสามารถจำกัดเวลาที่กลุ่มใช้งานอยู่ได้:

- **วันที่จะบล็อก**: เลือกวันที่กลุ่มจะใช้ วันที่ไม่ได้เลือกหมายความว่ากลุ่มไม่ได้ใช้งานในวันนั้น
- **กรอบเวลา**: รายการรูปแบบอิสระ หนึ่งหน้าต่างต่อบรรทัดในรูปแบบ `HHMM-HHMM` เช่น:

  ```
  0900-1000
  1200-1300
  ```

  กลุ่มใช้งานได้เฉพาะภายในหน้าต่างเหล่านั้นเท่านั้น รายการว่างหมายถึงทั้งวัน

สิ่งนี้ใช้ได้กับทุกประเภทกลุ่ม ยกเว้น `Custom` (กฎที่กำหนดเองสามารถใช้กำหนดการของตนเองได้โดยใช้ `ev.time.dayName` / `ev.time.hour` ดู **ส่วนที่ 11.4**)

## 8. แช่แข็ง (ป้องกันการงัดแงะ)

การแช่แข็งทำให้กลุ่มปิดการใช้งานได้ยากเมื่อมีแรงกระตุ้น

ในการ์ด **แช่แข็ง** ที่คุณเลือก:

- **หยุดนิ่ง** — คุณไม่สามารถแก้ไขหรือลบกลุ่มได้ และคุณไม่สามารถยกเลิกการเลือกปุ่มเปิดใช้งานได้ หากต้องการเปลี่ยนแปลงสิ่งใด คุณต้องดำเนินการพิธีกรรมละลายน้ำแข็ง (ดูด้านล่าง)
- **แช่แข็งแบบเข้มงวด** — เหมือนกับแช่แข็ง แต่จะยังคงล็อคเป็นเวลาหลายชั่วโมงที่คุณเลือก (ทศนิยม สูงสุด 72) จนกว่าเวลาจะหมดลง แม้แต่พิธีกรรมการละลายน้ำแข็งก็ยังไม่สามารถใช้งานได้

เมื่อปลดล็อคกลุ่มที่ถูกแช่แข็งได้ ปุ่ม **ยกเลิกการตรึง** จะปรากฏขึ้น การคลิกจะเป็นการเริ่ม **พิธีกรรม 20 ขั้นตอน**:

- กิริยาแสดงข้อความวินัยในตนเอง
- คุณต้องคลิก `Confirm` 20 ครั้ง
- มีการบังคับรอ 5 วินาทีระหว่างการคลิก
- หากคุณยกเลิก ณ จุดใด ๆ คุณต้องเริ่มต้นใหม่จากขั้นตอนที่ 1
- 20 ข้อความหมุนเวียนเพื่อให้คุณอ่านได้จริง

หากกลุ่มถูกทำเครื่องหมายว่า "ไม่เลื่อน" ด้วย (ดูหัวข้อถัดไป) คุณจะไม่สามารถเลื่อนได้ในขณะที่แช่แข็ง

สถานะการแช่แข็งจะแสดงอยู่ในบรรทัดเมตาของการ์ดกลุ่ม รวมถึงเวลาที่เหลือสำหรับการแช่แข็งแบบเข้มงวด

## 9. เลื่อน (ปิดใช้งานชั่วคราว)

เลื่อนปิดกลุ่มชั่วคราวโดยไม่ต้องยกเลิกการตรึง รองรับการเปิดใช้งานแบบล่าช้า คูลดาวน์หลังการเลื่อนซ้ำ ขั้นตอนการยืนยัน และระยะเวลารวมของการเลื่อนซ้ำ

ในการ์ด **เลื่อนการเตือน** ให้ทำดังนี้

- **อนุญาตให้กลุ่มนี้เลื่อนการปลุก** — หากปิด กลุ่มนี้จะไม่สามารถเลื่อนการปลุกได้เลย (รวมถึงขณะแช่แข็งด้วย)
- **เลื่อนเป็นเวลา (นาที)** — ทศนิยม ระยะเวลาในการเลื่อนปลุก
- **ความล่าช้าในการเปิดใช้งาน (นาที)** — ทศนิยม `>= 0` หลังจากที่คุณยืนยันการเลื่อน กลุ่มจะบล็อกต่อไปจนกว่าความล่าช้านี้จะผ่านไป จากนั้นการเลื่อนจึงจะเริ่มทำงาน
- **คูลดาวน์หลังจากเลื่อนปลุก (นาที)** — ทศนิยมจาก `0` ถึง `5` หลังจากการเลื่อนการเตือนเสร็จสิ้น คุณจะไม่สามารถเริ่มการเลื่อนการเตือนซ้ำอีกครั้งสำหรับกลุ่มนี้ได้จนกว่าคูลดาวน์จะสิ้นสุดลง
- **เวลายืนยัน** — จำนวนเต็ม `>= 0` หากเป็น `0` ระบบจะตั้งเวลาเลื่อนทันที มิฉะนั้น การเริ่มการเลื่อนปลุกจะเป็นการเปิดพิธีกรรมการยืนยันที่มีหลายขั้นตอนขนาดนั้น

ขั้นตอนการยืนยันการเลื่อนแต่ละขั้นตอนจะมีการบังคับ **รอ 5 วินาที** ก่อนที่จะอนุญาตให้คลิกครั้งถัดไป โมดอลจะบอกคุณอย่างชัดเจนและแสดงการนับถอยหลังแบบสดบนปุ่ม

หากกลุ่มถูกหยุด การตั้งค่าการเลื่อนจะถูกล็อคตามค่าที่เลือกก่อนการหยุด คุณยังสามารถเลื่อนการปลุกได้ ตราบใดที่อนุญาตให้เลื่อนการปลุกได้ แต่คุณต้องใช้การตั้งค่าการหน่วงเวลา / คูลดาวน์ / การยืนยันที่บันทึกไว้

การ์ดเลื่อนการเตือนยังแสดง **ระยะเวลาการเลื่อนทั้งหมด** สำหรับกลุ่มนั้นด้วย ยอดรวมนี้จะนับระยะเวลาการเลื่อนปลุกที่ใช้งานอยู่ทั้งหมด แม้ว่าไซต์จะสามารถเข้าถึงได้ด้วยเหตุผลอื่นบางประการในช่วงกรอบเวลานั้นก็ตาม

เมื่อการเลื่อนเสร็จสิ้น กฎจะกลับมาทันที หากกลุ่มไม่ได้ถูกระงับ ส่วนขยายจะค้างอีกครั้งโดยอัตโนมัติเมื่อสิ้นสุดการเลื่อน

ข้อความสถานะยืนยันการเลื่อนปลุก เมื่อการเลื่อนสิ้นสุดลง กลุ่มจะกลับสู่สภาวะปกติโดยอัตโนมัติ

คุณยังปิดการเลื่อนปลุกก่อนเวลาได้ด้วยปุ่ม **สิ้นสุดการเลื่อนปลุก**

สำหรับกลุ่มที่กำหนดเอง การกด **เริ่มเลื่อน** จะเป็นการส่งเหตุการณ์ `snoozePress` ไปยังกฎด้วย (ดูตารางเหตุการณ์ใน **ส่วนที่ 11**) ดังนั้นกฎที่กำหนดเองจึงสามารถบันทึกการกด บันทึกการให้เหตุผล หรือเริ่มการทำงานของเหตุการณ์ติดตามผลได้ กฎนี้ **ไม่มี API การเลื่อนการทำงานแบบเป็นโปรแกรม** โดยสามารถตอบสนองต่อสื่อได้ แต่ไม่สามารถยกเลิกหรือขยายได้

  - หากกลุ่มใดกลุ่มหนึ่งถูกระงับอย่างเข้มงวดและยังคงล็อคอยู่ **ลบทั้งหมด** จะถูกปิดใช้งาน

## 11. กลุ่มที่กำหนดเอง — การอ้างอิงตามเหตุการณ์ (v1.1+)

ตั้งแต่เวอร์ชัน 1.1 เป็นต้นไป กฎที่กำหนดเองจะ **ขับเคลื่อนด้วยเหตุการณ์** กฎของคุณไม่ใช่ฟังก์ชันต่อฮาร์ทบีทซึ่งค่าที่ส่งคืนจะบล็อกเพจอีกต่อไป แต่เนื้อหากฎจะเป็นสคริปต์ที่ **ลงทะเบียนตัวจัดการ** สำหรับเหตุการณ์เฉพาะ (การเปิดหน้า การเปลี่ยน URL ฮาร์ทบีทของหน้า เหตุการณ์ที่กำหนดเอง …) ตัวจัดการจะคงการลงทะเบียนไว้ตลอดการนำทางเพจและสวิตช์แท็บ และอยู่ภายใน **แซนด์บ็อกซ์นอกหน้าจอ** ที่มีอายุการใช้งานยาวนาน

เนื้อหาของกฎจะดำเนินการ **หนึ่งครั้งต่อการคลิกเรียกใช้** (หรือหนึ่งครั้งเมื่อเปิดใช้งานกลุ่มและมีแหล่งที่มาที่ใช้งานอยู่แล้ว) หากต้องการโหลดตัวจัดการอีกครั้ง ให้คลิก **เรียกใช้** ในตัวแก้ไข ป๊อปอัปจะแสดงการแจ้งเตือนเพื่อขอให้คุณโหลดหน้าที่เปิดอยู่แล้วซ้ำ เพื่อให้กฎใหม่มีผลใช้เช่นกัน

### 11.1 ลายเซ็นกฎ

```js
(event, helpers) => {
  // Register handlers here. This function is called exactly once
  // per Run click (or when the group is enabled).
}
```

สองข้อโต้แย้ง:

- `event` — **เหตุการณ์การลงทะเบียน** สำหรับกลุ่มนี้ ใช้เพื่อลงทะเบียน แทนที่ แสดงรายการ นับ หรือยกเลิกการลงทะเบียนตัวจัดการ และกับเหตุการณ์ที่กำหนดเองของ `post(...)`
- `helpers` — ชุดตัวช่วย (ดู **11.3**)

ฟังก์ชันนี้ **ไม่** คาดว่าจะส่งคืนค่า การตัดสินใจที่จะบล็อกหรืออนุญาตจะเกิดขึ้นในภายหลัง เมื่อมีเหตุการณ์เกิดขึ้นและหนึ่งในตัวจัดการที่ลงทะเบียนของคุณโทรไปที่ `ev.preventDefault()` และ/หรือ `ev.setResult(...)`

### 11.2 วงจรชีวิต

- **เรียกใช้** (ปุ่มต่อกลุ่มในตัวแก้ไข): ขั้นแรกกลไกจะล้างตัวจัดการทุกตัวที่ถูกแท็กด้วยกลุ่มนี้ก่อนหน้านี้ จากนั้นจึงเรียกใช้เนื้อหากฎอีกครั้งในแซนด์บ็อกซ์นอกหน้าจอ นี่เป็นวิธีเดียวที่จะลงทะเบียนใหม่ได้หลังจากแก้ไขแหล่งที่มาแล้ว
- **ปิดการใช้งานกลุ่ม**: ตัวจัดการทุกตัวที่แท็กกลุ่มนี้จะถูกล้าง แหล่งที่มาของกลุ่มจะถูกเก็บไว้ในที่เก็บข้อมูล แต่หยุดตอบสนองต่อเหตุการณ์
- **เปิดใช้งานกลุ่มอีกครั้ง**: กลไกจะเรียกใช้แหล่งที่มาที่ใช้งานอยู่สำหรับกลุ่มนี้อีกครั้งโดยอัตโนมัติ
- **ลบกลุ่ม**: เช่นเดียวกับการปิดการใช้งาน; ตัวจัดการทั้งหมดที่แท็กด้วยกลุ่มจะถูกล้าง
- **การลงทะเบียนใหม่ด้วย `(eventType, id)`**: จะแทนที่การลงทะเบียนก่อนหน้าอย่างเงียบๆ

แซนด์บ็อกซ์นอกหน้าจอแชร์โดยกลุ่มที่กำหนดเอง **ทั้งหมด** มีตัวจัดการจากกลุ่มต่างๆ อยู่ที่นั่น โดยแต่ละกลุ่มติดแท็กภายในด้วยรหัสกลุ่มของตนเอง เพื่อให้ "เรียกใช้" ปิดใช้งาน หรือลบเฉพาะกลุ่มที่ถูกต้องเท่านั้น

หากกฎที่กำหนดเองทำงานผิดปกติ (การวนซ้ำแบบซิงโครนัส สแปมบันทึกแบบรันอะเวย์ ฯลฯ) แซนด์บ็อกซ์จะกักกัน: กลุ่มจะถูกปิดใช้งานโดยอัตโนมัติและความล้มเหลวจะถูกบันทึกไว้เพื่อให้คุณเห็นได้ในแผงบันทึก หากต้องการเปิดใช้งานกฎที่ถูกกักกันอีกครั้ง ให้แก้ไขแหล่งที่มาแล้วคลิก **เรียกใช้** — กลไกจะล้างเหตุผลในการยกเลิกและโหลดกฎซ้ำ

### 11.2.1 การลงทะเบียนเหตุการณ์ (`event`)

วิธีการทั่วไป:

- `event.register(type, id, handler, options?)` - ลงทะเบียนตัวจัดการสำหรับประเภทเหตุการณ์ที่กำหนดเอง `id` เป็นทางเลือกของคุณเอง `options.priority` (ค่าเริ่มต้น `0`) — วิ่งสูงกว่าก่อน `options.intervalMs` — สำหรับ `tickEvent` เท่านั้น เค้นตัวจัดการเฉพาะนี้โดยสัมพันธ์กับเครื่องหมายโดยรวม การลงทะเบียนใหม่ด้วยการแทนที่ `(type, id)` เดียวกัน
- `event.unregister(type, id)`, `event.unregisterAll(type)`.
- `event.post(type, data?, { scope })` — เริ่มเหตุการณ์ที่กำหนดเอง `scope: "global"` เข้าถึงทุกกลุ่ม ค่าเริ่มต้น `scope: "group"` เข้าถึงตัวจัดการในกลุ่ม **เดียวกัน** เท่านั้น

น้ำตาลตามเหตุการณ์ (วิธีการหนึ่งชุดต่อประเภทบิวท์อิน):

- `event.registerTickEvent(id, handler, opts)`, `event.getTickEvent(id)`, `event.getTickEvents()`, `event.countTickRegistered()`.
- `event.registerOpenWebEvent(id, handler, opts)`, `event.getOpenWebEvent(id)`, `event.getOpenWebEvents()`, `event.countOpenWebRegistered()`.
- รูปร่างเดียวกันกับ `closeWebEvent`, `switchWebEvent`, `switchDomainEvent`, `webChangedEvent`, `pageHeartbeatEvent`, `timerEnded`, `snoozePress`.

### 11.2.2 ประเภทเหตุการณ์ในตัว

| พิมพ์ | เมื่อมันยิง | น้ำหนักบรรทุก `ev.data` |
|---|---|---|
| `tickEvent` | แชร์ขีด 1 วินาทีทั่วโลกทั่วทั้งเบราว์เซอร์ เริ่มทำงานโดยไม่คำนึงถึงการมองเห็นแท็บ ใช้สิ่งนี้สำหรับตรรกะแบบนาฬิกาที่ต้องทำงานต่อไปแม้ว่าจะไม่ได้โฟกัสแท็บก็ตาม | `{ intervalMs: 1000 }` |
| `pageHeartbeatEvent` | การเต้นของหัวใจ ~250 ms จากแท็บ **ใช้งานอยู่**, **มองเห็นได้** ขับเคลื่อนตรรกะการรับรู้การมองเห็นแท็บทั้งหมด รวมถึงการทำเครื่องหมายอัตโนมัติที่สร้างไว้ใน `getOrCreateTimer({ scope })` **ไม่** เริ่มทำงานจากแท็บพื้นหลังหรือในขณะที่หน้าจอถูกล็อค | `{ elapsedMs }` |
| `openWebEvent` | มีการสร้างแท็บใหม่หรือการนำทางใหม่ไปยัง URL ที่กลไกยังไม่เห็นสำหรับแท็บนั้น ไม่ **ไม่** เริ่มทำงานซ้ำสำหรับแท็บที่เปิดอยู่แล้วหลังจากการคลิกเรียกใช้ | `{ previousUrl, isNewTab }` |
| `closeWebEvent` | แท็บถูกปิด | `{ reason, nextUrl }` |
| `switchWebEvent` | URL **เปลี่ยนแปลง** ภายในแท็บเดียวกัน — ย้อนกลับ/ไปข้างหน้า การเปลี่ยนเส้นทาง SPA หรือการนำทางที่ลงสู่ URL ที่แตกต่างไปจากเมื่อก่อน **ไม่** เริ่มทำงานในการรีโหลดธรรมดา (URL เดียวกัน) | `{ previousUrl, previousHostname, sameDomain }` |
| `switchDomainEvent` | การเปลี่ยนแปลง URL ข้ามขอบเขตชื่อโฮสต์ (เช่น `youtube.com` → `wikipedia.org`) ยิงไปพร้อมกับ `switchWebEvent` | `{ previousUrl, previousHostname }` |
| `webChangedEvent` | เพจ (โหลดซ้ำ) ด้วยวิธีใดก็ตาม: เปิด, สลับ, อัปเดตประวัติ SPA, **หรือโหลดซ้ำธรรมดาที่เก็บ URL เดิม** นี่คือ hook "หน้าเปลี่ยนแปลง ประเมินใหม่ทุกอย่าง" ที่เชื่อถือได้ เริ่มทำงานควบคู่ไปกับ `openWebEvent` / `switchWebEvent` / `switchDomainEvent` และเป็นรายการเดียวที่เริ่มทำงานสำหรับการโหลด URL เดียวกันซ้ำ | `{ previousUrl, previousHostname, sameDomain, isFirstLoad, isReload, transition }` โดยที่ `transition` คือ `"tabCreated"`, `"commit"` หรือ `"history"` |
| `timerEnded` | ตัวจับเวลาที่จัดการโดยกลุ่มไปถึง `currentMs === 0` จัดส่งเฉพาะกลุ่มเจ้าของเท่านั้น | `{ timerId, displayName, direction, currentMs }` |
| `snoozePress` | ผู้ใช้กด **เริ่มเลื่อน** ในป๊อปอัปสำหรับกลุ่ม **กำหนดเอง** นี้ เหตุการณ์การแจ้งเตือนที่แท้จริง — ตัวจัดการสามารถเรียกใช้โค้ดที่กำหนดเองได้ (บันทึก เปลี่ยนเส้นทาง เริ่มเหตุการณ์อื่นๆ) แต่กฎที่กำหนดเองไม่มี **ไม่มี API การเลื่อนแบบเป็นโปรแกรม** ท่อนไม้ที่ผลิตที่นี่จะแสดงเป็นขนมปังปิ้งบนแท็บที่ใช้งานอยู่ จัดส่งเฉพาะกลุ่มกดเท่านั้น | `{ triggeredAt }` |

URL ใน `ev.url` และข้อมูลเหตุการณ์ **ทำให้เป็นมาตรฐาน** สำหรับเหตุการณ์: หน้าแท็บใหม่ของ โครม (ซึ่งแสดงผลพื้นผิว "ค้นหา Google หรือประเภท URL") ของ Google), `about:blank` และรูปแบบแท็บใหม่ที่เทียบเท่าจะแสดงเป็นสตริงว่าง `""` ดังนั้นตัวจับเวลาที่กำหนดขอบเขตไว้ที่ `ev.url === ""` จะทำเครื่องหมายเฉพาะเมื่อคุณอยู่ในหน้าแท็บใหม่เท่านั้น URL ปกติของ `google.com` จะไม่เปลี่ยนแปลง

### 11.2.3 วัตถุเหตุการณ์ (`ev`)

ตัวจัดการทุกตัวถูกเรียกใช้เป็น `(ev, helpers) => void` `ev` ประกอบไปด้วย:

- `ev.type` — ประเภทเหตุการณ์ที่จัดส่ง
- `ev.groupId` — รหัสกลุ่มผู้รับ
- `ev.tabId`, `ev.pageId`, `ev.url`, `ev.hostname` — บริบทสำหรับเหตุการณ์
- `ev.time` — สแน็ปช็อต `{ now, month, dayOfMonth, dayName, hour, minute }` เมื่อจัดส่ง `dayName` คือ `"Sunday"`..`"Saturday"`
- `ev.data` — เพย์โหลดเฉพาะเหตุการณ์ (ดูตารางด้านบน)

วิธีการ:

- `ev.preventDefault()` — ทำเครื่องหมายการจัดส่งว่า "ถูกบล็อก" สคริปต์เนื้อหาโฮสต์จะออกจากเพจ (หรือตาม `setRedirectLink`) เว้นแต่ว่าตัวจัดการที่มีลำดับความสำคัญสูงกว่าจะตั้งค่า `setResult(1)` ในภายหลัง
- `ev.stopPropagation()` — หยุดการจัดส่งนี้ทันที **ไม่มีการเรียกใช้ตัวจัดการเพิ่มเติมในกลุ่มใดๆ** สำหรับกิจกรรมนี้
- `ev.setResult(value)` — ตั้งค่าผลการจัดส่ง `value` อาจเป็น **หมายเลข** ใน `[-255, 255]` (บล็อก `-1`, `0` เป็นกลาง, `1` อนุญาต จำนวนเต็มอื่นๆ จะถูกสงวนไว้สำหรับตรรกะการแก้ไขข้อบกพร่องของคุณเอง) หรือ **สตริง** (ตีความว่าเป็น URL การเปลี่ยนเส้นทาง) การเรียก `setResult` ครั้งสุดท้ายกับตัวจัดการทั้งหมดจะชนะ ตัวเลข `1` จะแทนที่ `preventDefault` ก่อนหน้าใดๆ
- `ev.setRedirectLink(url)` / `ev.getRedirectLink()` — URL ที่โฮสต์ควรนำทางไปเมื่อการจัดส่งสิ้นสุดลงโดยถูกบล็อก นี่เป็นวิธีเดียวเท่านั้นที่จะเปลี่ยนเส้นทางจากกฎที่กำหนดเอง ตัวแก้ไขจะไม่เปิดเผยช่อง "URL เปลี่ยนเส้นทางเมื่อถูกบล็อก" สำหรับกลุ่มที่กำหนดเองอีกต่อไป
- `ev.post(type, data, { scope })` - ยิงเหตุการณ์ติดตามผลจากภายในตัวจัดการ

นอกจากนี้ `ev` ยังเป็น Proxy: ฟิลด์ใดๆ ที่คุณตั้งค่าไว้ (เช่น `ev.foo = 42`) จะถูกจัดเก็บไว้ในแมป `custom` และสามารถอ่านกลับได้จากตัวจัดการเดียวกันหรือจากตัวจัดการในภายหลังในการจัดส่งเดียวกัน### 11.3 วัตถุ `helpers`

การเรียกตัวจัดการทุกครั้งจะได้รับชุด `helpers` ใหม่ที่กำหนดขอบเขตเป็นกลุ่มที่รับและ URL ของเหตุการณ์ ฟิลด์คงที่:

- `helpers.now` — ยุคมิลลิวินาทีในการจัดส่ง
- `helpers.currentUrl` - URL เหตุการณ์ หลังจาก newtab/blank Normalization
- `helpers.groupId` — รับรหัสกลุ่ม

ทางลัดที่สะดวก (กำหนดเส้นทางไปยังฟังก์ชันการรับรู้ตัวสะสมเดียวกันกับที่ใช้โดยผู้ช่วยด้านล่าง ดังนั้นเอาต์พุตจึงยังคงอยู่ในแผงบันทึก):

- `helpers.log(...)`, `helpers.warn(...)`, `helpers.error(...)`.

วิธีการเข้าถึง:

- `helpers.getLogHelper()` — `log` / `warn` / `error`. เอาต์พุตมีการจำกัดอัตราและต่อยอดต่อการส่งเพื่อป้องกันไม่ให้กฎที่ควบคุมไม่ได้ค้างป๊อปอัป
- `helpers.getDomainHelper()` (นามแฝง `helpers.getDomainUtility()`) — การตรวจสอบ URL (ดู **11.3.5**)
- `helpers.getTimerHelper()` - ตัวจับเวลาที่กำหนดขอบเขตกลุ่ม (นับถอยหลัง / นับขึ้น) สถานะยังคงมีอยู่ตลอดการรีสตาร์ทเบราว์เซอร์
- `helpers.getPersistenceHelper()` — ที่เก็บคีย์/ค่า JSON กำหนดขอบเขตไว้ที่กลุ่ม
- `helpers.getRedirectionHelper()` — `setRedirectLink(url)` / `getRedirectLink()` (และนามแฝง `set` / `get`) บวกกับ `createMessageUrl(message)` ซึ่งส่งคืน URL `chrome-extension://...` ที่แสดงข้อความที่กำหนด
- `helpers.getPlatformHelper()` — เจตนา DOM ต่อแพลตฟอร์ม (ดู **11.3.6**)
- `helpers.getDOMHelper()` — เจตนา DOM ทั่วไป: `hide(sel)`, `show(sel)`, `addClass(sel, c)`, `removeClass(sel, c)`, `setText(sel, text)`, `click(sel)`, `injectCss(css, id?)`, `removeInjectedCss(id)`, `scrollTo(sel)`. การดำเนินการจะถูกจัดเป็นชุดและนำไปใช้หลังจากที่ตัวจัดการส่งคืน
- `helpers.getNavigationHelper()` — `back()`, `forward()`, `reload()`, `goTo(url)`, `closeTab()`. เอฟเฟกต์จะถูกนำไปใช้กับแท็บที่เกิดเหตุการณ์
- `helpers.getStorageHelper()` — ซุปเปอร์เซ็ตของ `getPersistenceHelper` บวก async `requestAsyncGet(key)` / `requestAsyncSet(key, value)` hooks สำหรับการจัดเก็บส่วนขยายข้าม (ผลลัพธ์มาถึงเป็นเหตุการณ์แบบกำหนดเองที่ตามมา)
- `helpers.getTabHelper()` — `list()`, `getActiveTab()`, `getById(id)`, `countOpen()` กับสแน็ปช็อตที่มาพร้อมกับกิจกรรม

วิธีการช่วยเหลือทั้งหมดปลอดภัย: พารามิเตอร์ที่ไม่ถูกต้องส่งคืน `null`, `false` หรือค่าว่างแทนที่จะส่งกลับ

#### 11.3.1 `getTimerHelper()`

ตัวจับเวลาต่อกลุ่ม ตัวจับเวลาแต่ละตัวจะถูกระบุด้วยสตริง `id` ที่คุณเลือก ข้อมูลประจำตัวถูกกำหนดขอบเขตไว้ที่กลุ่ม ดังนั้นทั้งสองกลุ่มจึงสามารถใช้รหัส `"yt-shorts"` ได้โดยไม่ขัดแย้งกัน สถานะยังคงมีอยู่ตลอดการรีสตาร์ทเบราว์เซอร์

สถานะคงอยู่ของตัวจับเวลาคือ: `id`, `displayName`, `direction` (`"forward"` หรือ `"backward"`), `isPaused` และ `currentMs` ไม่มี "ระยะเวลาเริ่มต้น" ที่เก็บไว้ — `isExpired` เป็นเพียง `currentMs === 0` ตัวจับเวลาการส่งต่อจะติ๊กตลอดไปและไม่มีวันหมดอายุด้วยตัวเอง ตัวจับเวลาถอยหลังหยุดฟ้องที่ `0` (ไม่มีค่าลบ)

มีสองวิธีในการก่อสร้าง เลือกอันที่ความหมายตรงกับสิ่งที่คุณต้องการ:

- `create({ id, displayName?, direction?, currentMs?, scope?, domain? })` — **สร้าง (สร้าง) ตัวจับเวลาใหม่เสมอ** ด้วยค่า init ที่ให้มา เขียนทับสถานะที่มีอยู่ รวมถึง `currentMs` ใช้สิ่งนี้เมื่อคุณหมายถึง "เริ่มต้นใหม่" เช่น ภายในสาขารีเซ็ตครั้งเดียว
- `getOrCreateTimer({ id, displayName?, direction?, currentMs?, scope?, domain? })` — **idempotent**. หากมีตัวจับเวลาด้วย `id` นั้นอยู่แล้ว `displayName` และ `direction` ของมันอาจถูกอัปเดต แต่ `currentMs` จะยังคงอยู่ มิฉะนั้นจะถูกสร้างขึ้นด้วยค่า init ที่ให้มา นี่คือสิ่งที่คุณต้องการสำหรับรูปแบบทั่วไป "ให้แน่ใจว่าตัวจับเวลาของฉันมีอยู่ แล้วปล่อยให้มันติ๊ก"

ทั้งสองวิธียอมรับฟังก์ชันเพรดิเคตสองฟังก์ชันที่กลไกจะจดจำตลอดอายุของกฎ (ฟังก์ชันเหล่านี้คงอยู่ตลอดช่วงการเต้นของหัวใจและการประเมินซ้ำของ `webChangedEvent` แต่จะ **ไม่เคยคงอยู่** ในการจัดเก็บ):- `scope: (url) => boolean` — เมื่อ `true` สำหรับ URL ที่มองเห็นได้ในปัจจุบันบน `pageHeartbeatEvent` แต่ละตัว ตัวจับเวลาจะทำเครื่องหมายอัตโนมัติตามช่วงการเต้นของหัวใจ (~ 250 ms) ผู้ช่วยเหลือไม่เคยปิดกั้น; มันอัพเดตเฉพาะ `currentMs` เท่านั้น ทำเครื่องหมายอัตโนมัติได้สูงสุดหนึ่งครั้งต่อการเต้นของหัวใจต่อตัวจับเวลา
- `domain: (url) => boolean` — เมื่อ `true` สำหรับ URL ที่มองเห็นได้ในปัจจุบัน ตัวจับเวลาจะแสดงผลในการซ้อนทับในเพจ (ซ้ายบน) เมื่อละเว้น `domain` เครื่องยนต์จะกลับไปที่ `scope` เพื่อแสดงผล ดังนั้นตัวจับเวลา "ติ๊กบน /shorts/ หน้า" จะปรากฏขึ้นที่นั่นโดยไม่มีการเดินสายเพิ่มเติม ระบุ `domain` อย่างชัดเจนหากคุณต้องการช่องแสดงผลอื่น (เช่น ทำเครื่องหมายเฉพาะบน `/shorts/` แต่แสดงเวลาที่เหลืออยู่ใน `youtube.com` ทั้งหมด)

> **ข้อสำคัญ — ตัวจับเวลาจะไม่บล็อกตัวเอง** เมื่อตัวจับเวลาถอยหลังถึงศูนย์ ตัวจับเวลาจะหยุดที่ศูนย์แล้วยิง `timerEnded` หนึ่งครั้ง การจะบล็อกเพจจริงหรือไม่นั้นขึ้นอยู่กับตัวจัดการ `openWebEvent` / `switchWebEvent` แยกต่างหากที่เรียก `ev.preventDefault()` หลังจากตรวจสอบ `helpers.getTimerHelper().isExpired(id)` แล้ว การแยกนี้ช่วยให้คุณสร้างตัวจับเวลา "คำเตือนเท่านั้น" ตัวติดตามการนับถอยหลัง การกระตุ้นแบบอ่อน หรือฮาร์ดบล็อก — แบบพื้นฐานเดียวกันที่คุณเลือกได้

วิธีการอื่นๆ:

- `delete(id)`, `pause(id)`, `resume(id)` — วงจรชีวิตมาตรฐาน หยุดการหยุดชั่วคราว `currentMs`
- `setDirection(id, "forward" | "backward")`, `setCurrentMs(id, ms)`, `addMs(id, deltaMs)` — มิวเทเตอร์โดยตรง (กฎส่วนใหญ่ไม่ต้องการสิ่งเหล่านี้ — ปล่อยให้การเต้นของหัวใจติ๊กตัวจับเวลาสำหรับคุณ)
- `setDisplayName(id, name)` — ติดฉลากใหม่
- `getCurrentMs(id)`, `getDirection(id)`, `getDisplayName(id)`, `isPaused(id)`, `exists(id)`.
- `isExpired(id)` — `true` หรือ `currentMs === 0`
- `getState(id)` — `{ id, displayName, direction, isPaused, currentMs, isExpired }` หรือ `null`
- `list()` — ทุกๆ ตัวจับเวลาที่กลุ่มนี้เป็นเจ้าของ เป็นอาร์เรย์ของวัตถุสถานะ

#### 11.3.2 `getPersistenceHelper()`

พื้นที่เก็บข้อมูลคล้ายแผนที่กำหนดขอบเขตให้กับกลุ่มของคุณ ค่าต้องเป็นอนุกรม JSON ได้

- `set(key, value)`, `get(key, defaultValue?)`, `has(key)`, `delete(key)`, `keys()`, `entries()`, `clear()`, `size()`.

ขีดจำกัดซอฟต์: ประมาณ 200 คีย์ต่อกลุ่ม 16 KB ต่อค่า

#### 11.3.3 `getLogHelper()`

- `log(...args)`, `warn(...args)`, `error(...args)` — เขียนไปยังแผง **บันทึก** ในป๊อปอัป (บันเดิลตัวช่วยยังคงกำหนดเส้นทางพวกมันผ่านตัวสะสมเดียวกันไม่ว่าการจัดส่งใดจะสร้างพวกมันก็ตาม) แต่ละบรรทัดขึ้นต้นด้วย `[CustomBlocker:groupId]`
- ตัวช่วยเหลือมีฮาร์ดแคป: ประมาณ **200 รายการบันทึกต่อการจัดส่ง** และความยาวสตริงสูงสุดต่อรายการ รายการที่เกินจะถูกทิ้งและนับใน `accumulator.logsDropped` นี่คือสิ่งที่ปกป้องป๊อปอัปจากการหลบหนี `for (let i = 0; i < 100000; i++) helpers.log(i)`
- เมื่อ **โหมดแก้ไขข้อบกพร่อง** ปิดอยู่ (ค่าเริ่มต้น) รายการระดับการติดตามที่เครื่องยนต์ปล่อยออกมา (การเริ่มจัดส่ง / เวลาตัวจัดการ) จะถูกระงับทุกที่ โดยจะไม่แสดงในแผงบันทึก และไม่พิมพ์ไปยังคอนโซล การโทร `log` / `warn` / `error` ของคุณเองจะผ่านเสมอ

#### 11.3.4 `getRedirectionHelper()`

ตรวจสอบ / แทนที่ URL การเปลี่ยนเส้นทางที่สคริปต์เนื้อหาจะใช้หากหน้าปัจจุบันถูกบล็อก

- `get()` — ส่งคืน URL การเปลี่ยนเส้นทางที่มีผลในปัจจุบันสำหรับการจัดส่งนี้ ในตอนแรกนี่คือ URL สำรองที่กำหนดค่าไว้ของกลุ่มในตัว (ถ้ามี) มิฉะนั้น `""`
- `set(url)` — แทนที่ URL ที่เปลี่ยนเส้นทางสำหรับการจัดส่งนี้ ส่งคืน `true` เมื่อสำเร็จ ส่งคืน `false` สำหรับอินพุตที่ไม่ใช่สตริง การส่งผ่าน `""` จะล้างการแทนที่การเปลี่ยนเส้นทางและถอยกลับไปเป็นพฤติกรรมการออกเริ่มต้นตามปกติ
- `createMessageUrl(message)` — ส่งคืน URL `chrome-extension://<id>/message-page.html?msg=...` ที่เมื่อนำทางไปแล้ว จะแสดงข้อความที่อยู่กึ่งกลางของหน้าที่สะอาด มีประโยชน์สำหรับการเปลี่ยนเส้นทางผู้ใช้ไปยังหน้าจอ "ไปทำงาน" / "หยุดพัก" หลังจากหมดเวลา ตัวอย่าง: `ev.setRedirectLink(h.getRedirectionHelper().createMessageUrl("Go Work"))`

เช่นเดียวกับผลข้างเคียงของกฎแบบกำหนดเองอื่นๆ สถานะนี้จะถูกแชร์กับกฎทั้งหมดในการจัดส่งปัจจุบัน เนื่องจากกฎทำงานจากล่างขึ้นบน กฎบนสุดในการเรียก `set(...)` จึงชนะ

#### 11.3.5 `getDomainHelper()` (นามแฝง `getDomainUtility()`)

ตัวช่วยตรวจสอบ URL ไม่มี `normalize()` เนื่องจาก URL ที่เข้ามาถูกทำให้เป็นแท็บมาตรฐานใหม่แล้ว

แกนหลัก:- `hostnameOf(url)`, `pathnameOf(url)`, `matches(hostname, site)`, `getPlatform(url)`.
- `isYouTubeHost`, `isTikTokHost`, `isInstagramHost`, `isFacebookHost`, `isTwitchHost`, `isRedditHost`, `isDiscordHost`.
- `youtube()`, `tiktok()`, `instagram()`, `facebook()`, `twitch()` — แต่ละรายการจะส่งกลับ `{ isPlatformUrl, isShortUrl, isVideoUrl, isPostUrl, isHomePage, extractAuthor, extractVideoId }`

การกรอง URL และตัวช่วยส่วน:

- `isEmptyStartPage(url)` — `true` สำหรับหน้าแท็บใหม่และรายการเทียบเท่า (URL ที่แสดงเป็น `""` ให้กับตัวจัดการ)
- `matchesAny(url, patterns)` — `patterns` อาจเป็น regex, regex สตริง หรืออาร์เรย์ของอย่างใดอย่างหนึ่ง
- `pathStartsWith(url, path)` — ทราบขอบเขต (`pathStartsWith("/r/", "/r")` เป็นจริง; `"/results/"` ไม่ใช่)
- `queryHas(url, key, value?)`, `queryGet(url, key)` — การตรวจสอบสตริงการสืบค้น
- `isSearchPage(url)` — จดจำผลลัพธ์ของ Google / Bing / DuckDuckGo / ยูทูบ / เรดดิต / ทวิตเตอร์ / X
- `isInfiniteFeedUrl(url)` — จดจำพื้นผิวฟีดอัลกอริธึมของ ยูทูบ, ติ๊กต็อก, อินสตาแกรม, เฟซบุ๊ก, เรดดิต, X
- `sameSection(a, b)` — ชื่อโฮสต์เดียวกันและส่วนเส้นทางแรกเหมือนกัน

#### 11.3.6 `getPlatformHelper()`

Intent DOM ต่อแพลตฟอร์มและตัวจับเวลาส่วนย่อย รวมถึงการตรวจสอบ `helpers.getPlatformHelper().<platform>()` แต่ละตัวจะส่งคืนอ็อบเจ็กต์ที่มีชุดวิธีการ **ปิดล้อมโดยแพลตฟอร์ม** — วิธีการที่ไม่สมเหตุสมผลบนแพลตฟอร์มที่กำหนดนั้นขาดหายไป ดังนั้นการเรียกพวกมันจะพ่น `TypeError: ... is not a function` แทนที่จะไม่มีการปฏิบัติการอย่างเงียบๆ ตัวอย่างเช่น ไม่มี `twitch().hidePosts` (ทวิช ไม่มีการโพสต์) และไม่มี `tiktok().hideShortButton` (ประสบการณ์ทั้งหมดของ ติ๊กต็อก มีวิดีโอแบบสั้นอยู่แล้ว _is_) ใช้ `helpers.getPlatformHelper().hasMethod(platform, name)` หรือ `.listMethods(platform)` เพื่อทบทวนขณะรันไทม์

> **อายุการใช้งานเพรดิเคตและกฎช่องเดียว** แต่ละ `hideShorts` / `hideReels` / `hideClips` / `hideStreams` / `hideVideos` / `hidePosts` / `filterComments` / `filterLive` เป็นเจ้าของ **หนึ่ง** เพรดิเคตถาวรต่อ `(group, platform, slot)`. เพรดิเคต **ไม่ได้** กำหนดขอบเขตไว้สำหรับเหตุการณ์ปัจจุบัน - เมื่อคุณตั้งค่าแล้ว เพรดิเคตจะยังคงใช้งานอยู่ในทุกการโหลดเพจและทุกการจัดส่งจนกว่าจะมีการเรียก `show*()` ที่ตรงกันหรือยกเลิกการโหลดกลุ่ม การเรียกเมธอดเดิมอีกครั้งด้วยฟังก์ชันใหม่ **แทนที่** อันก่อนหน้า — กลไกจะไม่รวมหรือรวมเพรดิเคตหลายรายการไว้ภายในกลุ่มเดียว ในการรวมเงื่อนไข ให้เขียนภาคแสดงหนึ่งตัวที่จะรวมตัวคุณเอง เช่น `yt.hideVideos(item => isShort(item) || hasKeyword(item))`. ข้ามกลุ่ม **ต่างกัน** แต่ละกลุ่มมีส่วนสนับสนุนภาคแสดงของตนเอง และรายการจะถูกซ่อนไว้หากภาคแสดงของกลุ่มใดๆ ตรงกัน

วิธีการตรวจสอบจะใช้ค่า ณ เวลาจัดส่งจากสแน็ปช็อตที่มาพร้อมกับเหตุการณ์ ความพร้อมใช้งานของพวกเขาถูกควบคุมโดยเมทริกซ์ด้านบน

ตัวแยกประเภท URL จะถูกเปิดเผยอีกครั้งเสมอโดยไม่คำนึงถึงแพลตฟอร์ม: `isPlatformUrl`, `isShortUrl`, `isVideoUrl`, `isPostUrl`, `isHomePage`, `extractAuthor`, `extractVideoId`ตัวจับเวลาส่วนย่อยจะลงทะเบียนตัวจับเวลาในบัคเก็ตกลุ่มถาวร และเมื่อกำหนดขอบเขตแล้ว จะทำเครื่องหมายเฉพาะ URL ที่ตรงกับส่วนย่อยนั้นเท่านั้น วิธีการจับเวลายอมรับ `{ id, direction, currentMs, displayName }` และติดตาม gating ต่อแพลตฟอร์มเดียวกัน

สำหรับวิธีการเพรดิเคต เพรดิเคตจะถูกเรียกต่อการ์ดที่ตรงกันกับ `item` ที่ทำให้เป็นมาตรฐาน: `{ url, name, author, length, views, publishedAt, description, live?, sponsored?, algorithmic? }` ฟิลด์ใดก็ได้สามารถเป็น `null`; "ไร้เดียงสาจนกว่าจะพิสูจน์ว่ามีความผิด" — ส่งคืน `false` เมื่อฟิลด์ที่คุณต้องการหายไป

### 11.4 ตัวอย่าง

**ง่าย** — บล็อกหน้า ยูทูบ Shorts ในช่วงเช้าวันธรรมดา:

```js
(event, helpers) => {
  const yt = helpers.getDomainHelper().youtube();

  function maybeBlock(ev) {
    if (!yt.isShortUrl(ev.url)) return;
    const { dayName, hour } = ev.time;
    const weekday = !["Saturday", "Sunday"].includes(dayName);
    if (weekday && hour >= 9 && hour < 12) {
      ev.preventDefault();
      ev.setResult(-1);
    }
  }

  event.registerOpenWebEvent("morning-block", maybeBlock);
  event.registerSwitchWebEvent("morning-block", maybeBlock);
}
```

**ปานกลาง** — งบประมาณรายวัน 30 นาทีสำหรับ ยูทูบ Shorts ตัวจับเวลาจะทำเครื่องหมายอัตโนมัติบน `pageHeartbeatEvent`s ในขณะที่มองเห็น URL ของวิดีโอสั้น ตัวจัดการที่แยกต่างหากจะบังคับใช้บล็อกเมื่อตัวจับเวลาถึงศูนย์

```js
(event, helpers) => {
  const TIMER_ID = "yt-shorts-budget";
  const yt = helpers.getDomainHelper().youtube();
  const onShorts = (url) => yt.isShortUrl(url);

  helpers.getTimerHelper().getOrCreateTimer({
    id: TIMER_ID,
    direction: "backward",
    currentMs: 30 * 60 * 1000,
    displayName: "YT Shorts",
    scope: onShorts,
    domain: onShorts
  });

  function maybeBlock(ev, h) {
    if (!yt.isShortUrl(ev.url)) return;
    if (h.getTimerHelper().isExpired(TIMER_ID)) {
      ev.setRedirectLink("https://example.com/focus");
      ev.preventDefault();
      ev.setResult(-1);
    }
  }
  event.registerOpenWebEvent("budget-block", maybeBlock);
  event.registerSwitchWebEvent("budget-block", maybeBlock);

  event.registerTimerEndedEvent("budget-warn", (_ev, h) => {
    h.getLogHelper().log("Budget hit zero.");
  });
}
```

**ยากขึ้น** — ซ่อนวิดีโอสั้นของ ยูทูบ แต่ละรายการที่มีการจัดการของผู้เขียนยาวเกินไป และแทรก CSS "วิดีโอสั้นนี้ถูกซ่อนไว้":

```js
(event, helpers) => {
  const MAX_AUTHOR_LEN = 16;

  function configure(_ev, h) {
    const yt = h.getPlatformHelper().youtube();
    yt.hideShorts(
      (item) => item.author && item.author.length > MAX_AUTHOR_LEN,
      { blockPageOnVisit: true }
    );
    h.getDOMHelper().injectCss(
      "ytd-rich-grid-media[data-cb-hidden] { opacity: 0.2 !important; }",
      "long-author-label"
    );
  }

  event.registerOpenWebEvent("hide-long-shorts", configure);
  event.registerSwitchWebEvent("hide-long-shorts", configure);
  event.registerWebChangedEvent("hide-long-shorts", configure);
}
```

**ยากที่สุด** — ถ่ายทอดเหตุการณ์ที่กำหนดเองจากผู้จัดการคนหนึ่งไปยังอีกคนหนึ่ง:

```js
(event, helpers) => {
  event.registerSwitchDomainEvent("track-domain", (ev) => {
    ev.post("domainChange", { from: ev.data.previousHostname, to: ev.hostname });
  });

  event.register("domainChange", "log-it", (ev, h) => {
    h.getLogHelper().log("crossed", ev.data.from, "→", ev.data.to);
  });
}
```

## 12. เทมเพลต

กลุ่มที่กำหนดเองแต่ละกลุ่มจะมีเครื่องมือเลือก **เทมเพลต** ที่เปิดเบราว์เซอร์ที่ตั้งค่าไว้ล่วงหน้าที่สามารถค้นหาได้ ขณะนี้ไลบรารีจัดส่ง **เทมเพลตมากกว่า 50 แบบ** โดยจัดเป็นเก้าหมวดหมู่ เพื่อให้คุณสามารถเรียกดูแทนการเขียนกฎตั้งแต่เริ่มต้น:

| หมวดหมู่ | ตัวอย่าง |
|---|---|
| **ตัวจับเวลา** | งบประมาณเวลาไซต์ (นับถอยหลัง + บล็อก), ตัวติดตามเวลาไซต์ (นับขึ้น), ขีดจำกัด ยูทูบ Shorts, ขีดจำกัดฟีด ติ๊กต็อก, ขีดจำกัดฟีด อินสตาแกรม, ขีดจำกัด เฟซบุ๊ก Reels, ขีดจำกัดคลิป ทวิช, งบประมาณเบี่ยงเบนความสนใจสากล, การติดตามการทำงานลึกรายวัน |
| **กำหนดการ** | บล็อกชั่วโมงทำงานในวันธรรมดา, ไซต์เฉพาะสุดสัปดาห์, ปิดก่อนเข้านอน, อนุญาตเพียงหนึ่งชั่วโมง, ข่าวเฉพาะอาหารกลางวัน, เริ่มต้นใหม่วันจันทร์, อนุญาต N นาทีแรกของแต่ละชั่วโมง, บล็อกที่เข้มงวดในการทำงานลึก |
| **ฟีด / กางเกงขาสั้น** | บล็อก URL กางเกงขาสั้นของ ยูทูบ, ซ่อนการ์ดกางเกงขาสั้น, ซ่อนกางเกงขาสั้นตามคำหลัก, ซ่อนฟีดหน้าแรกของ ยูทูบ / ความคิดเห็น / แนวโน้ม, บล็อก ติ๊กต็อก FYP, ซ่อนกางเกงขาสั้น ติ๊กต็อก, บล็อก URL วงล้อ อินสตาแกรม, ซ่อนฟีด อินสตาแกรม Reels, ซ่อนฟีด เฟซบุ๊ก / ม้วน, ซ่อน เรดดิต / ทวิตเตอร์ / หน้าแรกของ LinkedIn |
| **เปลี่ยนเส้นทาง** | สิ่งรบกวนใจ → หน้าโฟกัส, กางเกงขาสั้น → /feed/subscriptions, reddit.com → old.reddit.com, twitter / x → Nitter, แท็บใหม่ → รายการงาน |
| **โฟกัส** | เซสชันโฟกัสเฉพาะรายการที่อนุญาต Pomodoro 25/5 บล็อกระหว่างการประชุม บล็อกหลังจากการเข้าชม N วันนี้ บล็อกการสูญเสียสตรีค |
| **เขยิบ** | บันทึกการเข้าชมที่รบกวนสมาธิทุกครั้ง เตือนการเข้าชม Shorts แต่ละครั้ง นับการเข้าชมเว็บไซต์รายวัน |
| **ความคงอยู่** | ขีดจำกัดการเข้าชมรายเดือน, สลับการแบนรายสัปดาห์, ติดตามช่อง ดิสคอร์ด ที่เยี่ยมชม |
| **ปรับแต่ง DOM** | ซ่อนการสลับการเล่นอัตโนมัติของ ยูทูบ ซ่อน ทวิตเตอร์ / X "เกิดอะไรขึ้น" ทั่วไป "ซ่อนตัวเลือกบนไซต์" |
| **แก้ไขข้อบกพร่อง** | นับถอยหลังการสาธิต (3 วินาที) บันทึกทุกเหตุการณ์ที่กำหนดเอง |

ชิปกรองที่ด้านบนของตัวเลือกทำให้รายการแคบลงตามหมวดหมู่ (`Timer`, `Schedule`, `Feed`, …) และแพลตฟอร์ม (`ยูทูบ`, `ติ๊กต็อก`, `อินสตาแกรม`, …) การเลือกเทมเพลต:

1. โหลดอินพุตพารามิเตอร์ (URL, นาที, ช่วงชั่วโมง ฯลฯ) ลงในรูปแบบขนาดเล็ก
2. **ใช้ค่าที่ตั้งไว้ล่วงหน้า** เพื่อดูตัวอย่างแหล่งที่มาที่สร้างขึ้น
3. หลังจากยืนยัน **แทนที่กฎที่กำหนดเองปัจจุบันด้วยค่าที่ตั้งไว้ล่วงหน้านี้แล้ว** แหล่งที่มาจะถูกเขียนลงในโปรแกรมแก้ไข
4. จากนั้นคุณคลิก **เรียกใช้** เพื่อลงทะเบียนตัวจัดการกฎในแซนด์บ็อกซ์นอกหน้าจอ

เทมเพลตถูกกำหนดไว้ภายใต้ `templates/*.js` (`timers.js`, `schedule.js`, `feed.js`, …) แต่ละไฟล์เรียก `CB_REGISTER_TEMPLATES([...])` ณ เวลาโหลด และป๊อปอัปจะใช้รายการที่ผสาน การเพิ่มเทมเพลตใหม่หมายถึงการเขียนหนึ่งรายการลงในไฟล์ที่เหมาะสม โดยไม่ต้องใช้ระบบประปาอื่นๆ

## 13. พฤติกรรมหลายหน้า- แท็บที่เปิดอยู่ทั้งหมดในกลุ่มเดียวกันจะใช้ตัวจับเวลาเดียวกัน
- เมื่อคุณสลับไปที่แท็บในกลุ่มเดียวกัน การซ้อนทับของแท็บจะรีเฟรชทันทีเพื่อแสดงเวลาที่แชร์ในปัจจุบัน
- ตัวจับเวลากฎที่กำหนดเองทำเครื่องหมายเฉพาะบนแท็บ **มองเห็นได้ชัดเจน** — ขับเคลื่อนโดย `pageHeartbeatEvent` แท็บพื้นหลังและหน้าต่างย่อเล็กสุดไม่เลื่อนไปข้างหน้า ซึ่งตรงกับการนับถอยหลังกลุ่มบล็อกเริ่มต้น
- เมื่อมีการเพิ่มกฎใหม่ ทุกหน้าที่เปิดอยู่จะตรวจพบการเปลี่ยนแปลงและประเมินใหม่ภายในเสี้ยววินาที **แต่** ตัวจัดการที่ลงทะเบียนใหม่จะไม่ "เปิด" แท็บที่เปิดอยู่แล้วย้อนหลัง ป๊อปอัปแสดงการแจ้งเตือนการโหลดซ้ำหลังจากการเรียกใช้แต่ละครั้งด้วยเหตุผลดังกล่าว
- เมื่อกฎหมดอายุ การ์ดฟีดที่ซ่อนอยู่และปุ่มนำทางจะถูกกู้คืนในการรีเฟรชครั้งถัดไป

## 14. การตั้งค่า

เปิดกล่องโต้ตอบ **การตั้งค่า** ผ่านไอคอนรูปเฟืองในแถบด้านบน

- **ช่วงการเต้นของหัวใจ** — ความถี่ที่สคริปต์เนื้อหารายงานเวลาแท็บและขับเคลื่อน `pageHeartbeatEvent` ค่าเริ่มต้น 250 มิลลิวินาที ค่าที่ต่ำกว่าจะตอบสนองได้ดีกว่าแต่ใช้ CPU มากกว่า
- **ทำเครื่องหมายช่วง** — ความถี่ที่ `tickEvent` ทั่วโลกเริ่มทำงาน ค่าเริ่มต้น 1,000 มิลลิวินาที
- **โหมดแก้ไขข้อบกพร่อง** — *ปิด* ตามค่าเริ่มต้น เมื่อ *เปิด* กลไกจะส่งรายการระดับการติดตามไปยังแผงบันทึก (`[trace] dispatchEvent`, `[trace] N handler(s)`) และบรรทัด `[CustomBlocker:trace]` ไปยังคอนโซลของเบราว์เซอร์ ทิ้งมันไว้ในชีวิตประจำวัน เปิดใช้งานในขณะที่วินิจฉัยกฎที่ทำงานผิดปกติ `pageHeartbeatEvent` ถูกแยกออกจากการบันทึกการติดตามแม้ว่าจะเปิดโหมดแก้ไขข้อบกพร่องอยู่ก็ตาม เนื่องจากมีการดำเนินการสี่ครั้งต่อวินาทีและจะทำให้ส่วนที่เหลือจมหายไป

## 15. ความเป็นสากล

UI ทั้งหมดได้รับการแปลแล้ว ใช้เครื่องมือเลือก **ภาษา** ที่มุมขวาบน

ภาษาที่รองรับ ได้แก่ อังกฤษ จีน (ตัวย่อ) สเปน ญี่ปุ่น เกาหลี รวมถึงครอบคลุมบางส่วนสำหรับภาษาฮินดี อาหรับ เบงกาลี โปรตุเกส รัสเซีย ปัญจาบ เยอรมัน ฝรั่งเศส ตุรกี เวียดนาม อิตาลี ไทย ดัตช์ โปแลนด์ อินโดนีเซีย อูรดู และเปอร์เซีย ภาษาที่มีการครอบคลุมบางส่วนจะกลับไปเป็นภาษาอังกฤษหากไม่มีสตริงที่ขาดหายไป

คู่มือการใช้งานจะโหลดไฟล์มาร์กดาวน์ที่ตรงกับภาษาที่คุณเลือก โดยมีภาษาอังกฤษเป็นทางเลือก

## 16. ข้อความสถานะ

ข้อความสถานะจะปรากฏเป็นข้อความปิ้งตรงกลางซึ่งจะจางหายไปหลังจากผ่านไปประมาณสองวินาที:

- "บันทึกการเปลี่ยนแปลง"
- "สร้าง \"ชื่อกลุ่ม\" แล้ว"
- "โหลดกฎแบบกำหนดเองแล้ว — มีตัวจัดการ N ตัวทำงานอยู่ หากต้องการใช้กฎนี้กับแท็บที่คุณเปิดไว้ ให้โหลดซ้ำ"
- ข้อผิดพลาดในการตรวจสอบความถูกต้อง เช่น "นาทีที่อนุญาตต้องเป็นตัวเลขที่มากกว่า 0"

สำหรับช่องป้อนข้อมูลที่มีข้อกำหนดด้านรูปแบบ ข้อความจะแสดงถัดจากปุ่มที่เกี่ยวข้องด้วย (สำหรับการเลื่อนซ้ำ)

## 17. ความเป็นส่วนตัวและการจัดเก็บ

- ทุกอย่างถูกจัดเก็บไว้ใน `chrome.storage.local` ไม่มีข้อมูลถูกส่งไปที่ใดก็ได้
- รายการที่จัดเก็บได้แก่: กลุ่มของคุณ ตัวจับเวลาการใช้งาน เวลารีเซ็ตครั้งล่าสุด บันทึกการงีบหลับ ตัวจับเวลาแบบกำหนดเอง และค่าถาวรแบบกำหนดเอง
- ส่วนขยายจะไม่อ่านเนื้อหาของหน้าเกินกว่าที่จำเป็นในการตรวจจับประเภทหน้า (เส้นทาง / ชื่อโฮสต์ / เครื่องหมาย DOM ที่รู้จักสำหรับไซต์วิดีโอ) และเพื่อประเมินเพรดิเคตที่ผู้ใช้เขียน จะไม่อ่านข้อความ โพสต์ ความคิดเห็น หรือเนื้อหาส่วนตัวของคุณ

## 18. การอนุญาต

- `storage` — สำหรับข้อมูลข้างต้น
- `declarativeNetRequest` — สำหรับการบล็อกดั้งเดิมของกลุ่ม `Default`
- `alarms` — เพื่อกำหนดเวลาการเปลี่ยนกฎอย่างมีประสิทธิภาพ
- `tabs`, `webNavigation` — เพื่อตรวจจับการสร้างแท็บ การเปลี่ยนแปลง URL และฮาร์ทบีทของหน้า เพื่อให้สามารถส่งกิจกรรมได้
- `offscreen` — เพื่อโฮสต์แซนด์บ็อกซ์กฎแบบกำหนดเองที่มีอายุการใช้งานยาวนาน
- `host_permissions: <all_urls>` — ดังนั้นสคริปต์เนื้อหาจึงสามารถแสดงการซ้อนทับของตัวจับเวลาและตรวจจับบริบทของแพลตฟอร์มในหน้าใดก็ได้

## 19. การแก้ไขปัญหา- **กลุ่มที่ฉันเพิ่มไม่ได้ทำอะไรเลย** ตรวจสอบให้แน่ใจว่ากลุ่มเปิดใช้งานอยู่ ตามกำหนดการที่อนุญาตในขณะนี้ ไม่มีการเลื่อนการทำงาน และ (สำหรับกลุ่มแพลตฟอร์ม) หน้าเว็บตรงกับประเภทเนื้อหาที่เลือกและตัวกรองผู้เขียนจริงๆ
- **ตัวจับเวลาค้างหรือไม่ถูกต้องบนแท็บเดียว** สลับออกไปและย้อนกลับ หรือโฟกัสที่แท็บ ซึ่งจะทำให้เกิดการรีเฟรชแบบบังคับจากตัวจับเวลาที่แชร์
- **การ์ดฟีดปรากฏขึ้นอีกครั้งหลังจากที่ฉันคิดว่าควรซ่อนไว้** การซ่อนฟีดจะทำงานเฉพาะในขณะที่กฎกำลังบล็อกอยู่เท่านั้น หากคุณมีกฎ `after-minutes` ให้ป้อนการซ่อนเตะเข้าเมื่อเวลาของคุณเหลือศูนย์
- **ปุ่มนำทางของ ยูทูบ ที่ฉันคาดว่าจะซ่อนไว้ยังคงอยู่ที่นั่น** การซ่อนการนำทางจำเป็นต้องตั้งค่ากฎเป็น "อย่ากรองตามผู้เขียน" และประเภทเนื้อหาต้องเป็นวิดีโอสั้นหรือโพสต์ ยูทูบ ด้วยตัวกรองผู้เขียน การซ่อนจะกระทำต่อการ์ดเท่านั้น
- **กฎที่กำหนดเองไม่ได้ทำอะไรเลยหรือโยนทิ้งไปอย่างเงียบๆ** เปิดการตั้งค่า → เปิดใช้งาน **โหมดแก้ไขข้อบกพร่อง** จากนั้นคลิก **เรียกใช้** อีกครั้งและดูแผงบันทึก บรรทัดที่นำหน้าด้วย `[trace]` จะแสดงทุกการจัดส่งและตัวจัดการ ใช้ `helpers.getLogHelper().log(...)` เพื่อเพิ่มจุดติดตามของคุณเอง หากกฎที่ทำงานผิดปกติยังคงถูกกักกันอัตโนมัติ ให้แก้ไขแหล่งที่มาแล้วคลิกเรียกใช้ — เรียกใช้เพื่อล้างเหตุผลที่ยกเลิก
- **กฎที่กำหนดเองใหม่ของฉันไม่มีผลกับแท็บที่เปิดอยู่แล้ว** โหลดซ้ำ กฎที่กำหนดเองแนบกับกิจกรรมหน้า *ในอนาคต* ป๊อปอัปจะแสดงการเตือนให้โหลดซ้ำหลังจากการวิ่งทุกครั้ง
- **ตัวจับเวลาถอยหลังของฉันไม่เดินหน้า** ตัวจับเวลากฎที่กำหนดเองทำเครื่องหมายที่แท็บ **มองเห็นได้ชัดเจน** ผ่านทาง `pageHeartbeatEvent` เท่านั้น แท็บพื้นหลัง หน้าต่างที่ย่อเล็กสุด และหน้าจอที่ถูกล็อคจะหยุดการทำงานชั่วคราวตามการออกแบบ ซึ่งเป็นพฤติกรรมเดียวกับการนับถอยหลังเริ่มต้นของกลุ่มบล็อก
- **ฉันไม่สามารถลบกลุ่มได้** มันอาจจะค้างอยู่ กลุ่มที่แช่แข็งแบบเข้มงวดไม่สามารถลบได้เลยจนกว่าการล็อคจะหมดอายุ กลุ่มที่แช่แข็งแบบไม่เข้มงวดสามารถลบออกได้ผ่านพิธีการยกเลิกการตรึง
- **ป๊อปอัปแสดงข้อความ "กำลังทำงาน…" ตลอดไป** กฎที่กำหนดเองอาจวนซ้ำอย่างแน่นหนา เครื่องยนต์ดับลงหลังจากหมดเวลาอย่างหนักและกักกันกฎ เปิดแผงบันทึกด้วยเหตุผลการยกเลิก แก้ไขกฎแล้วคลิกเรียกใช้

## 20. อภิธานศัพท์

- **กลุ่มบล็อก** — กฎหนึ่งชุดที่มีประเภท พฤติกรรม กำหนดการ และการหยุด/เลื่อนเป็นของตัวเอง
- **บล็อกทันที** — กฎจะบล็อกทันทีทุกครั้งที่มีการใช้งาน
- **การบล็อกหลังนาที** — กฎจะเริ่มการบล็อกหลังจากงบประมาณเวลาสำหรับช่วงเวลานั้นหมดลงเท่านั้น
- **รีเซ็ตช่วงเวลา** — ความถี่ในการรีเซ็ตงบประมาณหลังนาทีที่
- **กำหนดการ** — วัน + กรอบเวลาที่กลุ่มใช้งานอยู่
- **การแช่แข็ง / การแช่แข็งอย่างเข้มงวด** — สถานะป้องกันการปลอมแปลง
- **เลื่อน** — ปิดใช้งานชั่วคราวด้วยพิธีกรรมการยืนยันที่กำหนดค่าได้
- **ตัวกรองผู้เขียน** — สำหรับกลุ่มแพลตฟอร์ม จำกัดกฎไว้เฉพาะผู้สร้างเนื้อหาบางราย
- **ประเภทเนื้อหา** — สำหรับกลุ่มแพลตฟอร์ม จำกัดกฎไว้เฉพาะเนื้อหาบางรูปแบบ (สั้น ยาว โพสต์)
- **ตัวช่วยเหลือ** — ยูทิลิตี้ที่ส่งผ่านไปยังตัวจัดการกฎที่กำหนดเอง
- **แพลตฟอร์ม** — หนึ่งใน `youtube`, `tiktok`, `facebook`, `instagram`, `twitch` แต่ละกลุ่มมีประเภทกลุ่มและตรรกะการซ่อนฟีดของตัวเอง
- **การเต้นของหัวใจ** — ~250 ms `pageHeartbeatEvent` ส่งจากแท็บที่มองเห็นได้ซึ่งทำงานอยู่
- **ติ๊ก** — `tickEvent` ที่แชร์ทั่วโลกของ 1 (ไม่สามารถมองเห็นได้)
- **โหมดแก้ไขข้อบกพร่อง** — การตั้งค่าที่แสดงการบันทึกการติดตามภายในในแผงบันทึกและคอนโซลของเบราว์เซอร์
- **กักกัน** — ปิดใช้งานกฎที่กำหนดเองโดยอัตโนมัติซึ่งเกินขีดจำกัดความปลอดภัยของรันไทม์ (กำหนดเวลา บันทึกสแปม …) เคลียร์ในการวิ่งครั้งถัดไป

## 21. ข้อจำกัด- การซ่อนฟีดขึ้นอยู่กับ DOM ปัจจุบันของแต่ละแพลตฟอร์ม หากแพลตฟอร์มเปลี่ยนเค้าโครง อาจจำเป็นต้องอัปเดตตัวเลือกการซ่อน
- การตรวจจับบริบทของแพลตฟอร์มสำหรับไซต์ที่ไม่ใช่ ยูทูบ ส่วนใหญ่จะอิงตาม URL ดังนั้นจึงเชื่อถือได้มากที่สุดสำหรับ URL เนื้อหาตามรูปแบบบัญญัติ
- ตัวจับเวลากฎที่กำหนดเองทำเครื่องหมายที่ความละเอียดฮาร์ทบีท (~ 250 ms) อย่าพึ่งพาสิ่งเหล่านี้ในช่วงเวลาเสี้ยววินาที
- เพรดิเคตที่ส่งผ่านไปยัง `hideShorts` / `hideVideos` / `hidePosts` ได้รับการประเมินพร้อมกันต่อการ์ดฟีด ตรรกะที่หนักหน่วงในภาคแสดงอาจทำให้การเลื่อนฟีดช้าลง เก็บไว้ราคาถูก
- สองแท็บที่แก้ไขตัวจับเวลาต่อกลุ่มเดียวกันใช้กลยุทธ์ "การเขียนครั้งสุดท้ายชนะ" ไปพร้อมๆ กัน สำหรับการใช้งานทั่วไปก็ถือว่าใช้ได้ หากคุณขึ้นอยู่กับการบัญชีที่แน่นอน คาดว่าจะมีการคลาดเคลื่อนเล็กน้อยเป็นครั้งคราว
- เบราว์เซอร์อาจระงับพนักงานบริการพื้นหลังเมื่อไม่ได้ใช้งาน ส่วนขยายจะกลับมาทำงานต่อทันทีที่เพจหรือการเตือนต้องการ งบประมาณการใช้งานไซต์/ตามกำหนดเวลาจะนับผ่านการเล่นซ้ำฮาร์ทบีท
