# การอ้างอิงการทำงานของแอปเดสก์ท็อป Vault

## วัตถุประสงค์และขอบเขต

นี่เป็นข้อมูลอ้างอิงที่เชื่อถือได้สำหรับอินเทอร์เฟซแอปพลิเคชันเดสก์ท็อปห้องนิรภัย โดยเจตนาจะแยกจากคู่มือส่วนขยายเบราว์เซอร์ของห้องนิรภัย

แอปเดสก์ท็อปจัดการ **แอปพลิเคชันดั้งเดิมและหน้าต่างแอปพลิเคชัน** ส่วนขยายเบราว์เซอร์จัดการเว็บไซต์ แท็บเบราว์เซอร์ และฟีดแพลตฟอร์มเว็บที่รองรับ พวกเขาแบ่งปันแนวคิดเดียวกัน ได้แก่ กลุ่ม กำหนดการ ตัวจับเวลา การค้าง การเลื่อนเวลา กฎที่กำหนดเอง และบริดจ์เสริม แต่ไม่ได้มีรูปแบบการบังคับใช้ที่เหมือนกัน

ใช้เอกสารนี้เพื่อกำหนดค่า ตรวจสอบ ทำซ้ำ หรือรักษาลักษณะการทำงานของแอปเดสก์ท็อป รหัสนี้เป็นที่ยอมรับหากการใช้งานและคู่มือนี้แตกต่างออกไป

## 1. สิ่งที่แอปเดสก์ท็อปสามารถควบคุมได้และไม่สามารถควบคุมได้

ห้องนิรภัยจะประเมินนโยบายโฟกัสสำหรับแอปพลิเคชันเนทีฟที่เลือก เมื่อความสามารถในการบังคับใช้ดั้งเดิมพร้อมใช้งาน ก็สามารถใช้แผนปัจจุบันกับเป้าหมายแอปพลิเคชันที่ตรงกัน และรายงานผลการป้องกัน/สถานะไปยัง UI ของโฮสต์

มันสามารถ:

- สร้าง เปิดใช้งาน ปิดใช้งาน เรียงลำดับใหม่ นำเข้า ส่งออก ตรึง เลื่อน และลบกลุ่ม
- กำหนดเป้าหมายแอปพลิเคชันดั้งเดิมที่เลือกผ่านตัวเลือกแอปพลิเคชัน
- ใช้การบล็อกทันที ระยะเวลาเผื่อเวลา หรือตัวจับเวลาแบบนับขึ้นเท่านั้น
- จำกัดกลุ่มปกติไว้เฉพาะวันธรรมดาและหน้าต่างเวลาท้องถิ่น
- รันกฎนโยบาย JavaScript แบบกำหนดเองสำหรับเหตุการณ์วงจรชีวิตของแอปพลิเคชัน
- แสดงสถานะดั้งเดิม/ข้อมูลพาเนลที่สร้างกฎผ่านโฮสต์
- จัดการโฟลเดอร์ภายในเครื่องที่เป็นตัวเลือกสำหรับคำขอไฟล์กฎแบบกำหนดเองที่รองรับ
- เข้าร่วมกลุ่มที่เชื่อมโยงอย่างชัดเจนที่เข้ากันได้ผ่านสะพาน Vault ในเครื่อง

ไม่สามารถ:

- ทำหน้าที่เป็นส่วนขยายของเบราว์เซอร์ ตรวจสอบ DOM ของเว็บไซต์ หรือจัดการการ์ดฟีดของเบราว์เซอร์
- รับประกันว่าระบบปฏิบัติการจะอนุญาตให้ทุกแอปพลิเคชัน กระบวนการ หน้าต่าง หรือบริการของระบบสามารถควบคุมได้
- เปลี่ยนการเลือกแอปพลิเคชันเป็นการดูแลระบบระยะไกล การเฝ้าระวังอุปกรณ์ หรือไฟร์วอลล์
- ทำให้ตัวช่วยแบบกำหนดเองเฉพาะเบราว์เซอร์ เช่น DOM, การนำทาง, การเปลี่ยนเส้นทาง หรือการควบคุมแท็บทำงานในรันไทม์ดั้งเดิม
- ซิงโครไนซ์ทุกกลุ่มโดยอัตโนมัติเพียงเพราะบริดจ์ในเครื่องกำลังทำงานอยู่

| แผนการบังคับใช้ | การตัดสินใจอนุญาต/โล่/สถานะที่เป็นผลลัพธ์ของโฮสต์ดั้งเดิมหลังจากประเมินกลุ่มที่เกี่ยวข้อง |

## 3. ข้อมูลประจำตัวเป้าหมายและตัวเลือกแอปพลิเคชัน

เลือกแอปพลิเคชันผ่านตัวเลือก **+** ในกลุ่มแอปพลิเคชันเริ่มต้น ห้องนิรภัยจัดเก็บข้อมูลประจำตัวที่เป็นมาตรฐานและชื่อที่แสดง

| โฮสต์ | ข้อมูลประจำตัวเป้าหมายที่ใช้สำหรับการจับคู่ |
| --- | --- |
| macOS | ตัวระบุบันเดิลแอปพลิเคชัน หากมี |
| หน้าต่าง | พาธปฏิบัติการหรือชื่อกระบวนการที่ทำให้เป็นมาตรฐานซึ่งจัดทำโดยตัวเลือกแอปพลิเคชัน |

ชื่อที่แสดงเป็นชื่อสำหรับเอดิเตอร์ ค่าที่ทำให้เป็นมาตรฐานคือข้อมูลประจำตัวที่ใช้โดยเลเยอร์การบังคับใช้ดั้งเดิม การเปลี่ยนชื่อแอปพลิเคชันใน UI จะไม่เปลี่ยนข้อมูลระบุตัวตน เป้าหมายยังมีแท็กสำหรับการใช้นโยบายกฎที่กำหนดเองอีกด้วย

อย่าป้อน URL เว็บไซต์ในช่องเป้าหมายของแอปพลิเคชัน และคาดว่าจะมีการบังคับใช้แอปพลิเคชันดั้งเดิม ใช้กลุ่มไซต์ของส่วนขยายสำหรับการบล็อกเว็บไซต์

## 4. วงจรชีวิตกลุ่มและลำดับความสำคัญ

กลุ่มใหม่จะถูกเปิดใช้งานตามค่าเริ่มต้น รายการกลุ่มรองรับการเลือก เปิด/ปิด ลำดับการลาก เพิ่ม ล้าง นำเข้า ส่งออก และการลบ กลุ่มที่เลือกจะเปิดขึ้นในตัวแก้ไข

การแก้ไขฟิลด์ปกติจะบันทึกผ่านนโยบายการบันทึกอัตโนมัติของตัวแก้ไข กลุ่มที่ถูกแช่แข็งจะปิดใช้งานการควบคุมการแก้ไขแบบธรรมดา แหล่งที่มาแบบกำหนดเองแตกต่างออกไป: การบันทึกข้อความไม่ได้ทำให้ใช้งาน **เรียกใช้** คือการดำเนินการที่โหลดแหล่งที่มาปัจจุบันลงในรันไทม์ของนโยบาย

หลายกลุ่มสามารถจับคู่แอปพลิเคชันเดียวกันได้ ห้องนิรภัยจะประเมินนโยบายกลุ่มตามลำดับที่เก็บไว้และสร้างแผนการบังคับใช้ดั้งเดิม จงตั้งใจให้การทับซ้อนโดยเฉพาะอย่างยิ่งเมื่อกลุ่มใช้นโยบายที่มีกำหนดเวลาต่างกันหรืออนุญาต/ป้องกันปัญหากฎที่กำหนดเอง จัดลำดับกลุ่มใหม่เพื่อให้ลำดับความสำคัญที่ตั้งใจไว้ชัดเจน ไม่ต้องพึ่งพาการกำหนดค่าที่ขัดแย้งกันซึ่งได้รับการแก้ไขด้วยวิธีที่เป็นมิตรต่อผู้ใช้โดยเฉพาะ

## 5. กลุ่มแอปพลิเคชันปกติ

### 5.1 สถานะกลุ่ม

| สนาม | สัญญาการทำงาน |
| --- | --- |
| ชื่อ | ไม่ว่างเปล่า ตัดขอบ และไม่คำนึงถึงตัวพิมพ์เล็กและตัวพิมพ์ใหญ่ภายในจุดสิ้นสุดนี้ |
| เปิดใช้งาน | กลุ่มผู้พิการจะยังคงอยู่ แต่ไม่ได้มีส่วนร่วมในการบังคับใช้ตามปกติ |
| เป้าหมาย | ข้อมูลระบุตัวตนของแอปพลิเคชันอย่างน้อยหนึ่งรายการที่เลือกจากตัวเลือก |
| พฤติกรรม | บล็อกทันที บล็อกหลังจากเผื่อไว้ หรือจับเวลา/นับถอยหลัง |
| กำหนดการ | วันธรรมดาที่เลือกและกรอบเวลาท้องถิ่นที่เป็นตัวเลือก |
| ตรึง | ไม่มี, แช่แข็ง, แช่แข็งแบบเข้มงวด หรือ แช่แข็งโดยผู้ปกครอง |
| เลื่อน | นโยบายข้อยกเว้นชั่วคราวต่อกลุ่ม |
| ข้อความสำรอง/สถานะ | ข้อความที่โฮสต์ดั้งเดิมสามารถแสดงได้เมื่อใช้การตอบกลับแบบโล่/สถานะ |

กลุ่มเริ่มต้นที่ว่างเปล่าไม่มีเป้าหมายแอปพลิเคชันที่เลือก ดังนั้นจึงไม่ตรงกับแอปพลิเคชันที่มีอยู่เท่านั้น

### 5.2 การปิดกั้นพฤติกรรม

| พฤติกรรม | ผลลัพธ์ |
| --- | --- |
| บล็อกทันที | เป้าหมายที่แอ็คทีฟที่ตรงกันจะสร้างการตัดสินใจแบบบล็อก/ชีลด์ทันที |
| บล็อกหลังจากผ่านไปหลายนาที | การใช้งานที่ตรงกันจะเกิดขึ้นกับค่าเผื่อกลุ่ม เมื่อค่าเผื่อหมดลง กลุ่มจะสร้างการตัดสินใจแบบบล็อก/ชีลด์ดั้งเดิมจนกว่าระยะเวลาการใช้งานจะรีเซ็ตหรือสถานะอื่นทำให้กลุ่มไม่ทำงาน |
| จับเวลา (นับถอยหลัง ไม่มีการบล็อก) | การใช้การจับคู่จะถูกวัดและอาจแสดงขึ้น แต่ตัวจับเวลานั้นเพียงอย่างเดียวจะไม่สร้างบล็อก |

กลุ่มใหม่ใช้เวลาเผื่อไว้ 15 นาทีและช่วงเวลารีเซ็ต 24 ชั่วโมง เว้นแต่จะมีการเปลี่ยนแปลง การใช้งานตามกำหนดเวลาเป็นของกลุ่ม ดังนั้นเป้าหมายที่ตรงกันทั้งหมดจึงแชร์นโยบายกลุ่มนั้น การตอบสนองที่แน่นอนต่อบล็อกจะดำเนินการโดยโฮสต์ดั้งเดิม และถูกจำกัดโดยสิทธิ์ของระบบปฏิบัติการและกลไกการบังคับใช้ที่รองรับ

### 5.3 กำหนดการ

กำหนดการใช้กับกลุ่มปกติ กลุ่มที่กำหนดเองทำการตัดสินใจเรื่องเวลาของตนเองใน JavaScript

เลือกชุดค่าผสมระหว่างวันจันทร์ถึงวันอาทิตย์ แต่ละกรอบเวลาจะมีหนึ่งบรรทัดในเวลาท้องถิ่น:

```text
0900-1200
1330-1730
```

รูปแบบที่ยอมรับคือ HHMM-HHMM ชั่วโมงต้องเป็น 00 ถึง 23 นาที 00 ถึง 59 และการเริ่มต้นต้องมาก่อนจุดสิ้นสุดในวันเดียวกัน หน้าต่างรวมจุดเริ่มต้นและไม่รวมจุดสิ้นสุด ไม่ยอมรับหน้าต่างข้ามเวลาเที่ยงคืน หน้าต่างว่างหมายถึงทั้งวันที่เลือก

กลุ่มปกติจะทำงานเฉพาะเมื่อ:

1. เปิดใช้งานอยู่;
2. เลือกวันทำงานปัจจุบัน;
3. เวลาท้องถิ่นอยู่ภายในหน้าต่างที่กำหนดไว้ หรือกลุ่มไม่มีหน้าต่าง
4. ไม่ได้อยู่ในช่วงงีบหลับที่ใช้งานอยู่

### 5.4 เลื่อนการเตือน

เลื่อนจะลบกลุ่มปกติออกจากการบังคับใช้ที่ใช้งานอยู่ชั่วคราว ระยะของมันคือ:

| เฟส | ผลลัพธ์ |
| --- | --- |
| รอดำเนินการ | มีคำขออยู่แต่ความล่าช้าในการเปิดใช้งานยังไม่ผ่านไป กลุ่มยังคงทำงานอยู่ |
| ใช้งานอยู่ | กลุ่มนี้ไม่ได้ใช้งานชั่วคราวเนื่องจากมีการเลื่อนการเตือน |
| คูลดาวน์ | การเลื่อนการปลุกสิ้นสุดลงแล้ว และกลุ่มกลับมาใช้งานได้อีกครั้ง แต่การเลื่อนการเตือนใหม่ยังไม่พร้อมใช้งาน |

| การตั้งค่า | กฎ |
| --- | --- |
| อนุญาตให้เลื่อน | เมื่อปิดแล้ว จะไม่สามารถเลื่อนกลุ่มได้ตามปกติ |
| ระยะเวลาเลื่อน | จำนวนนาทีที่เป็นบวก ค่าเริ่มต้นสำหรับกลุ่มใหม่คือ 30 นาที |
| การเปิดใช้งานล่าช้า | ศูนย์นาทีหรือมากกว่านั้นก่อนที่จะเริ่มการเลื่อนปลุก |
| คูลดาวน์ | ศูนย์ถึงห้านาทีหลังจากการงีบหลับที่ใช้งานอยู่สิ้นสุดลง |
| การยืนยัน | การโต้ตอบการยืนยันที่จำเป็นจำนวนเต็มไม่เป็นลบ |

การเลื่อนการแจ้งเตือนที่ใช้งานอยู่ถือเป็นข้อยกเว้นของนโยบายชั่วคราว ไม่ใช่การลบหรือยกเลิกการหยุดการทำงาน การกำหนดค่ากลุ่มยังคงเหมือนเดิม

### 5.5 หยุดนิ่ง

การแช่แข็งเป็นอุปสรรคในการปรับเปลี่ยนโดยเจตนา

| โหมด | สัญญา |
| --- | --- |
| แช่แข็ง | การแก้ไขทั่วไปและการเปลี่ยนแปลงสถานะปกติจะยังคงล็อกอยู่จนกว่าขั้นตอนการยืนยันการยกเลิกการตรึงของผลิตภัณฑ์จะสำเร็จ |
| แช่แข็งอย่างเข้มงวด | ไม่สามารถเลิกตรึงกลุ่มได้ก่อนที่ระยะเวลาการแช่แข็งแบบเข้มงวดจะสิ้นสุดลง ระยะเวลาเป็นค่าบวกและจำกัดไว้ที่ 72 ชั่วโมง |
| ผู้ปกครองถูกระงับ | จำเป็นต้องมีการจัดการรหัสผ่านผู้ปกครองสำหรับการหยุด/ยกเลิกการตรึง |

การเลือกโหมดในตัวแก้ไขจะไม่หยุดกลุ่มด้วยตัวเอง ใช้การหยุดเพื่อนำไปใช้ กลุ่มที่เชื่อมโยงกับบริดจ์อาจล็อกการควบคุมการหยุดการประสานงานในขณะที่สมาชิกที่ต้องการออฟไลน์

## 6. การบังคับใช้ดั้งเดิมและการควบคุมอุปกรณ์

ตัวแก้ไขสามารถบันทึกกลุ่มได้อย่างถูกต้องแม้ว่าระบบปฏิบัติการจะไม่อนุญาตให้บังคับใช้ก็ตาม ตรวจสอบ **การตั้งค่า → การควบคุมอุปกรณ์** และสถานะเนทิฟสดทุกครั้งหลังจากเปลี่ยนการอนุญาต

โฮสต์ดั้งเดิมจะตัดสินใจว่าการดำเนินการใดที่เป็นไปได้สำหรับระบบปฏิบัติการ แอปพลิเคชัน หน้าต่าง และสถานะการอนุญาตในปัจจุบัน สามารถกำหนดค่ากฎได้อย่างถูกต้องแต่จะไม่มีผลกระทบที่มองเห็นได้เมื่อ:

- การควบคุมอุปกรณ์ไม่ได้รับอนุญาตหรือถูกเพิกถอน
- กลุ่มถูกปิดใช้งาน กำหนดเวลาออก หรือเลื่อนการแจ้งเตือนชั่วคราว
- กระบวนการที่โฟกัสไม่ตรงกับเป้าหมายมาตรฐานที่เลือก
- ระบบปฏิบัติการปฏิเสธการดำเนินการสำหรับเป้าหมายนั้น
- การพึ่งพาบริดจ์ออฟไลน์สำหรับการดำเนินการที่ต้องมีสถานะการประสานงาน

อย่าถือว่าการบันทึกขนมปังสำเร็จเป็นข้อพิสูจน์ว่าการบังคับใช้ที่ใช้งานอยู่มีอยู่ ทดสอบเป้าหมายที่เลือกในขณะที่กลุ่มทำงานอยู่ และตรวจสอบสถานะโฮสต์

## 7. กลุ่มที่กำหนดเองและกฎนโยบายดั้งเดิม

กลุ่มที่กำหนดเองทำงานในรันไทม์นโยบาย JavaScript ดั้งเดิม ไม่ใช่กฎที่กำหนดเองของเบราว์เซอร์ DOM ของเบราว์เซอร์ แท็บ การนำทาง การเปลี่ยนเส้นทาง URL และพฤติกรรมการควบคุมฟีดไม่สามารถใช้งานได้โดยไม่ได้ตั้งใจ

### 7.1 วงจรชีวิตของแหล่งที่มา

Use a function expression:

```js
(events, helpers) => {
  events.on("focusEvent", "shield-focus", (event, h) => {
    if (event.target?.id) event.setResult(-1);
  });
}
```

Run loads the source and its event registrations. Running again unloads the old source and resets its rule-owned handlers, timers, panels, persistence, and dynamic app blocklist. A source that does not evaluate to a function cannot be loaded.

### 7.2 เหตุการณ์ในตัวแบบเนทีฟ

| เหตุการณ์ | ความหมาย |
| --- | --- |
| ติ๊กเหตุการณ์ | เห็บโฮสต์เป็นระยะ ตัวเลือกการลงทะเบียน IntervalMs สามารถจำกัดอัตราตัวจัดการได้ |
| ตัวจับเวลาสิ้นสุด | การนับถอยหลังที่เป็นของกฎจะถึงศูนย์ |
| งีบหลับกด | ผู้ใช้กด Start Snooze สำหรับกลุ่มแบบกำหนดเอง |
| แผงเหตุการณ์ | มีการใช้การควบคุมแผงแบบกำหนดเอง |
| localFileEvent | การดำเนินการโฟลเดอร์ในเครื่องที่ร้องขอเสร็จสมบูรณ์ |
| openAppEvent | แอปพลิเคชันที่ถูกติดตามจะเปิดขึ้น |

| MinimizeEvent / ไม่ย่อเล็กสุดEvent | โฮสต์รายงานการเปลี่ยนแปลงแบบย่อหน้าต่างที่รองรับ |

| แอพ ChangedEvent | เหตุการณ์วงจรการใช้งาน/การเปลี่ยนแปลงทั่วไปของแอปพลิเคชัน |

ออบเจ็กต์เหตุการณ์ประกอบด้วยประเภท, groupId/groupID, groupName, URL/ชื่อโฮสต์ที่เทียบเท่า, เวลา, ข้อมูล และเป้าหมาย สำหรับแอปพลิเคชันแบบเนทีฟ เป้าหมายจะแสดงรหัส ชนิด ชื่อจอแสดงผล ค่าที่ทำให้เป็นมาตรฐาน และแท็กเมื่อเป้าหมายโฟกัสตรงกับเป้าหมายที่กำหนดค่าไว้

ข้อมูลเหตุการณ์วงจรชีวิตของแอปพลิเคชันประกอบด้วยรหัส/ชื่อแอปปัจจุบัน ชื่อกลุ่ม สแน็ปช็อตแอปที่รันอยู่เป็นอนุกรม และค่าเฉพาะเหตุการณ์ เช่น BundleId, PreviousAppId, currentAppId หรือเหตุผลในการเปลี่ยนแปลง

### 7.3 Event API และการตัดสินใจ

รีจิสทรีจัดเตรียมเปิด/ลงทะเบียน, ปิด/ยกเลิกการลงทะเบียน, ยกเลิกการลงทะเบียนทั้งหมด, countRegistered, getEvent และ getEvents ลำดับความสำคัญที่สูงกว่าจะทำงานก่อน ลำดับความสำคัญเท่ากันจะรักษาลำดับการลงทะเบียน รีจิสทรีมีตัวจัดการสูงสุดต่อกลุ่ม

วัตถุเหตุการณ์รองรับ:

| วิธีการ | ผลลัพธ์ |
| --- | --- |
| setResult(-1) | สร้างการตัดสินใจแบบโล่/บล็อกแบบเนทีฟ ผลลัพธ์สตริงจะกลายเป็นบล็อกดั้งเดิมเนื่องจากกฎเดสก์ท็อปไม่มีเป้าหมายการเปลี่ยนเส้นทางของเบราว์เซอร์ |
| อนุญาต (เหตุผล) หรือ setResult(1) | สร้างการตัดสินใจอนุญาตสำหรับเหตุการณ์ |
| setShieldMessage(ข้อความ) | ตั้งค่าโล่/ข้อความสถานะที่หันหน้าเข้าหามนุษย์สำหรับบล็อกเนทิฟ |
| หยุดการขยายพันธุ์() | หยุดตัวจัดการในภายหลังสำหรับเหตุการณ์ปัจจุบัน |
| บล็อก (appId), ปลดบล็อก (appId) | เพิ่ม/ลบบล็อกแอปพลิเคชันดั้งเดิมแบบไดนามิก |
| ปิด (appId), เปิด (appId) | ขอการดำเนินการปิด/เปิดดั้งเดิมที่รองรับ |
| โพสต์(ประเภท, ข้อมูล) | ส่งเหตุการณ์ที่กำหนดเองที่ซ้อนกันภายในรันไทม์ดั้งเดิม |

รันไทม์ของแอปอนุญาตให้ใช้ตัวจับเวลา การคงอยู่ แผง บันทึก การดำเนินการของโฟลเดอร์ในเครื่อง ตัวช่วยหน้าต่างแอปพลิเคชัน และยูทิลิตีตัวแยกประเภท URL โดยจงใจปฏิบัติต่อ DOM การนำทาง การเปลี่ยนเส้นทาง และผู้ช่วยเหลือแท็บเบราว์เซอร์ว่าไม่พร้อมใช้งาน/เฉื่อย

| --- | --- |
| getLogHelper | ส่งการตัดสินใจบันทึกแอป/ป๊อปอัป/หน้าจอ |
| getTimerHelper | สร้างตัวจับเวลาเดินหน้า/ถอยหลังพร้อมขอบเขต ขั้นตอน ขอบเขต/ภาคแสดงโดเมน หยุดชั่วคราว/ดำเนินการต่อ การตรวจสอบสถานะ และการเปลี่ยนตัวจับเวลาที่สิ้นสุด ตัวจับเวลาไม่ได้ป้องกันด้วยตัวเอง |

## 8. โฟลเดอร์ไฟล์ในเครื่อง

โฟลเดอร์ไฟล์ในเครื่องเป็นขอบเขตทางเลือกที่ผู้ใช้อนุญาตสำหรับกฎที่กำหนดเอง กฎสามารถขอการอ่าน เขียน ผนวก รายการ การทดสอบการดำรงอยู่ และการดำเนินการ JSON ของข้อความ/CSV/JSON เส้นทางจะสัมพันธ์กับรูทที่เลือกเสมอ เส้นทางสัมบูรณ์ ส่วนการแวะผ่าน ส่วนประกอบเส้นทางที่ซ่อนอยู่ ส่วนขยายที่ไม่สนับสนุน และการดำเนินการภายนอกรากจะถูกปฏิเสธ

เพิกถอนโฟลเดอร์เมื่อกฎไม่ต้องการอีกต่อไป กฎต้องจัดการสิทธิ์ที่ไม่พร้อมใช้งานและผลลัพธ์ localFileEvent ที่ล้มเหลว จะต้องไม่ถือว่าโฟลเดอร์ที่เลือกยังคงได้รับอนุญาตหลังจากรีสตาร์ท

## 9. บริดจ์เว็บแอป

บริดจ์เป็นตัวเลือกการซิงโครไนซ์ภายในเครื่องระหว่างโปรแกรม Vault ที่เข้ากันได้ แอพเดสก์ท็อปดั้งเดิมสามารถโฮสต์ฮับท้องถิ่นได้ ไคลเอนต์เชื่อมต่อกับที่อยู่ในท้องถิ่นที่รองรับ

สถานะการเชื่อมต่อคือ ปิด กำลังเชื่อมต่อ ตัดการเชื่อมต่อ เชื่อมต่อ/ทำงานอยู่ และเกิดข้อผิดพลาด การเชื่อมต่อโปรแกรมไม่รวมกลุ่มทั้งหมด ผู้ใช้จะต้องเชื่อมโยงกลุ่มที่ตรงกันที่มีสิทธิ์อย่างชัดเจน

สำหรับลิงค์กลุ่ม:

1. เริ่มเนทิฟฮับในการตั้งค่า
2. เชื่อมต่อปลายทาง Vault อื่นที่เข้ากันได้
3. สร้างกลุ่มที่ตรงกันและเลิกตรึงด้วยชื่อและประเภทเดียวกัน
4. ในส่วนของ Group Bridge ให้เลือกโปรแกรมและเชื่อมต่อกลุ่ม

กลุ่มที่เชื่อมโยงกันจะก่อตัวเป็นคลัสเตอร์ ค่านโยบายทั่วไป การใช้งาน และสถานะการเลื่อนที่สนับสนุนสามารถซิงโครไนซ์ในขณะที่สมาชิกเชื่อมต่ออยู่ การยกเลิกการเชื่อมต่อจะหยุดการซิงโครไนซ์ชั่วคราวและรักษากลุ่มโลคัลไว้ เป้าหมายเฉพาะเบราว์เซอร์ การดำเนินการแบบกำหนดเองที่ไม่รองรับ และช่องเฉพาะแพลตฟอร์มไม่รับประกันว่าจะถ่ายโอนได้

## 10. นำเข้า ส่งออก รีเซ็ต และตรวจสอบ

การส่งออกจะบันทึกการแสดงกลุ่มที่เข้ากันได้หนึ่งรายการ นำเข้าตรวจสอบ/ทำให้ข้อมูลกลุ่มที่เข้ากันได้เป็นมาตรฐาน และยังคงบังคับใช้เอกลักษณ์ของชื่อท้องถิ่น ลบกลุ่ม ลบกลุ่มที่เลือกและสถานะที่เกี่ยวข้อง ล้างลบกลุ่มทั้งหมดหลังจากการยืนยัน การรีเซ็ตเป็นค่าเริ่มต้นส่งผลต่อการตั้งค่าตัวแก้ไขส่วนกลาง ส่งออกสิ่งใด ๆ ที่ต้องเก็บรักษาไว้ก่อน

ก่อนที่จะใช้กฎเดสก์ท็อป:

1. ตรวจสอบว่าได้รับการควบคุมอุปกรณ์แล้ว
2. ตรวจสอบข้อมูลประจำตัวที่ทำให้เป็นมาตรฐานของเป้าหมายที่เลือก
3. ตรวจสอบสถานะที่เปิดใช้งาน กำหนดการ สถานะหยุด และระยะเลื่อน
4. ทดสอบพฤติกรรมทันที หมดเวลา และนับถอยหลังแยกกัน
5. สำหรับกลุ่มแบบกำหนดเอง ให้เรียกใช้แหล่งที่มาที่แน่นอนและทดสอบเหตุการณ์แอพที่ลงทะเบียนแต่ละรายการ
6. ตรวจสอบความล้มเหลวของโฟลเดอร์ในเครื่องและการดำเนินการที่ประสบความสำเร็จ
7. ตรวจสอบพฤติกรรมการเชื่อมต่อแบบออฟไลน์/การเชื่อมต่อหากกลุ่มเชื่อมโยงอยู่

## 11. หมายเหตุเฉพาะแพลตฟอร์ม

มีการแชร์แนวคิดนโยบายหลัก แต่การบังคับใช้ดั้งเดิมนั้นเฉพาะโฮสต์:

| macOS | หน้าต่าง |
| --- | --- |
| โดยปกติแล้วเป้าหมายจะแก้ไขเป็นตัวระบุชุดแอปพลิเคชัน การควบคุมอุปกรณ์และการบังคับใช้เกตสถานะสิทธิ์อนุญาต macOS ปัจจุบัน | โดยปกติแล้วเป้าหมายจะแก้ไขเป็นพาธปฏิบัติการหรือชื่อกระบวนการที่ทำให้เป็นมาตรฐาน เลเยอร์การบังคับใช้ Windows จะตัดสินใจว่าหน้าต่าง/กระบวนการใดในปัจจุบันที่สามารถจัดการได้ |
